4 Information requirements

This clause specifies the information requirements for the Fabrication requirement application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Fabrication_requirement_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Fabrication_requirement_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Fabrication_technology_arm;    --  ISO/TS 10303-1670
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Fabrication_technology_arm ISO/TS 10303-1670

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 metal_or_dielectric   

A metal_or_dielectric lists the values for metal or dielectric.

EXPRESS specification:

*)
TYPE metal_or_dielectric = ENUMERATION OF
   (over_finished_conductor,
    over_finished_dielectric);
END_TYPE;
(*

Enumerated item definitions:

over_finished_conductor: specifies that value is over finished conductor;

over_finished_dielectric: specifies that value is over finished dielectric;

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Fabrication_thickness_requirement   

A Fabrication_thickness_requirement is a type of Predefined_requirement_view_definition that characterizes the interconnect thickness range over metallic material or dielectric material and assigns a tolerance to that range for fabrication purposes.

EXPRESS specification:

*)
ENTITY Fabrication_thickness_requirement
  SUBTYPE OF (Predefined_requirement_view_definition);
  characterized_class : metal_or_dielectric;
  characterized_class_range : Range_characteristic;
  tolerance : Tolerance_characteristic;
WHERE
  WR1: SELF\Product_view_definition.initial_context.life_cycle_stage = 'test';
  WR2: NOT EXISTS(SELF\Product_view_definition.id);
END_ENTITY;
(*

Attribute definitions:

characterized_class: specifies the metal_or_dielectric for the Fabrication_thickness_requirement. This is the class to which the range and tolerance is applied.

characterized_class_range: specifies a role of the Range_characteristic for the Fabrication_thickness_requirement. The characterized_class_range specifies the measurements to which the tolerance applies.

tolerance: specifies a role of the Tolerance_characteristic for the Fabrication_thickness_requirement. The tolerance is applicable over the range specified by the characterized_class_range.

Formal propositions:

WR1: The Product_view_definition.initial_context.life_cycle_stage shall be 'test'.

WR2: The id shall not be populated.

4.3.2 Layout_land_width_tolerance_requirement   

A Layout_land_width_tolerance_requirement is a type of Predefined_requirement_view_definition that characterizes the land width range that is required to be produced for a particular terminal pitch class of mating components.

EXPRESS specification:

*)
ENTITY Layout_land_width_tolerance_requirement
  SUBTYPE OF (Predefined_requirement_view_definition);
  mating_feature_pitch_class : Pitch_class;
  maximum_negative_deviation : Length_data_element;
WHERE
  WR1: SELF\Product_view_definition.initial_context.life_cycle_stage = 'test';
END_ENTITY;
(*

Attribute definitions:

mating_feature_pitch_class: specifies a role of the Pitch_class for the Layout_land_width_tolerance_requirement.

maximum_negative_deviation: specifies the largest negative Length_data_element allowed width deviation for the Layout_land_width_tolerance_requirement. The deviation is measured at the surface of the Land that mates with the external device.

NOTE    The measurement location deviates from normal fabrication practice of measuring at the base of the Land in order to guarantee successful assembly process.

Formal propositions:

WR1: The Product_view_definition.initial_context.life_cycle_stage shall be 'test'.

4.3.3 Layout_line_width_tolerance_requirement   

A Layout_line_width_tolerance_requirement is a type of Predefined_requirement_view_definition that characterizes the line width range that is expected by the design organization to be manufactured for a particular Stratum_technology.

EXPRESS specification:

*)
ENTITY Layout_line_width_tolerance_requirement
  SUBTYPE OF (Predefined_requirement_view_definition);
  characterized_stratum_technology : Stratum_technology;
  tolerance_line_width_class : Line_width_class;
  maximum_negative_deviation : Length_data_element;
  maximum_positive_deviation : Length_data_element;
WHERE
  WR1: SELF\Product_view_definition.initial_context.life_cycle_stage = 'test';
END_ENTITY;
(*

Attribute definitions:

characterized_stratum_technology: specifies a role of the Stratum_technology for the Layout_line_width_tolerance_requirement.

tolerance_line_width_class: specifies a role of the Line_width_class for the Layout_line_width_tolerance_requirement.

maximum_negative_deviation: specifies the largest negative Length_data_element for the Layout_line_width_tolerance_requirement.

maximum_positive_deviation: specifies the largest positive Length_data_element for the Layout_line_width_tolerance_requirement.

Formal propositions:

WR1: The Product_view_definition.initial_context.life_cycle_stage shall be 'test'.

4.3.4 Line_width_class   

A Line_width_class is a category which assigns a name to a length range for classification purposes. The Value_with_unit used as the basis for the range shall be a Length_data_element.

EXAMPLE    The Line_width_class may be exchanged so that tolerances may be applied to all members of the same class, and specified only once. E.g., all lines with a design width greater than 50 mils and less than 75 mils are in the 'big' class. All members of 'big' shall have a tolerance of +5/-5 mils. Other Application Objects must be populated in order to apply the tolerances to the class.

EXPRESS specification:

*)
ENTITY Line_width_class;
  class_member : Range_characteristic;
  class_name : STRING;
WHERE
  WR1: SIZEOF(QUERY(np <* class_member\Representation.items | NOT ('VALUE_WITH_UNIT_EXTENSION_ARM.LENGTH_DATA_ELEMENT' IN TYPEOF(np)))) = 0;
END_ENTITY;
(*

Attribute definitions:

class_member: specifies a role of the Range_characteristic for the Line_width_class. The Range_characteristic shall provide length data.

class_name: specifies a string for the Line_width_class.

Formal propositions:

WR1: The members of items of class_member shall be Length_data_elements.

4.3.5 Pitch_class   

A Pitch_class is a category in which a particular group of Part_feature may fall, depending on the value of their paired spacing. The Pitch_class controls the width of the mating feature which is measured perpendicular to the primary axis of a terminal in the class. The only Value_with_unit type permitted in the range is a Length_data_element.

EXAMPLE    The Pitch_class is included so that tolerances may be applied to all members of the same class, and specified only once. E.g., all pitchs with a design spacing greater than 50 mils and less than 75 mils are in the 'big' class.

NOTE    Other Application Objects will be populated in order to apply the class property to tolerances of the product being designed .

EXPRESS specification:

*)
ENTITY Pitch_class;
  class_member : Range_characteristic;
  class_name : STRING;
WHERE
  WR1: SIZEOF(QUERY(np <* class_member\Representation.items | NOT ('VALUE_WITH_UNIT_EXTENSION_ARM.LENGTH_DATA_ELEMENT' IN TYPEOF(np)))) = 0;
END_ENTITY;
(*

Attribute definitions:

class_member: specifies a role of the Range_characteristic for the Pitch_class. The range specifies the spacings which are included as a member of the Pitch_class. The Range_characteristic shall use Length_data_element for the Numerical_item_with_unit.

class_name: specifies a string for the Pitch_class.

Formal propositions:

WR1: The members of items of class_member shall be Length_data_elements.

4.4 ARM subtype constraint definition

This subclause specifies the ARM subtype constraint for this module. The subtype constraint places a constraint on the possible super-type / subtype instantiations. The ARM subtype constraint and definition is specified below.

4.4.1 fr_predefined_requirement_view_definition_subtypes   

The fr_predefined_requirement_view_definition_subtypes constraint specifies a constraint that applies to instances of subtypes of Predefined_requirement_view_definition.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT fr_predefined_requirement_view_definition_subtypes FOR Predefined_requirement_view_definition;
  ONEOF (Layout_line_width_tolerance_requirement,
         Layout_land_width_tolerance_requirement,
         Fabrication_thickness_requirement);
END_SUBTYPE_CONSTRAINT;
