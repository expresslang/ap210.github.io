4 Information requirements

This clause specifies the information requirements for the Composite constituent shape application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Composite_constituent_shape_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Composite_constituent_shape_arm;

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

The boundary_class_zone type allows for the designation of the data types Boundary_curve_representation and Loop.

EXPRESS specification:

*)
TYPE boundary_class_zone = SELECT
   (Boundary_curve_representation,
    Loop);
END_TYPE;

4.2.2 composite_assembly_shape   

The composite_assembly_shape type allows for the designation of the data types Advanced_brep_shape_representation, Edge_based_wireframe_shape_representation, Faceted_brep_shape_representation, Geometrically_bounded_surface_shape_representation, Geometrically_bounded_wireframe_shape_representation, Manifold_surface_shape_representation, Shell_based_wireframe_shape_representation and Three_d_geometry_set.

EXPRESS specification:

*)
TYPE composite_assembly_shape = SELECT
   (Advanced_brep_shape_representation,
    Edge_based_wireframe_shape_representation,
    Faceted_brep_shape_representation,
    Geometrically_bounded_surface_shape_representation,
    Geometrically_bounded_wireframe_shape_representation,
    Manifold_surface_shape_representation,
    Shell_based_wireframe_shape_representation,
    Three_d_geometry_set);
END_TYPE;
(*

4.2.3 core_shape

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Beveled_sheet_representation   

A Beveled_sheet_representation is a type of Constituent_shape_representation that has a significant thickness, and whose edge is beveled at a constant angle around its entire perimeter. It is an extension of the concept of a thin sheet to products, such as honeycomb core, that have significant thickness but can be formed to mate with a curved surface.

EXPRESS specification:

*)
ENTITY Beveled_sheet_representation
  SUBTYPE OF (Constituent_shape_representation);
  bevel_angle : Angle_data_element;
  boundary_surface : Composite_sheet_representation;
  vertical_profile_height : Length_data_element;
END_ENTITY;
(*

Attribute definitions:

bevel_angle: specifies the angle from the plane normal to the defining surface of part's profile to the beveled surface.

boundary_surface: specifies the base exterior boundary and defining surface of the part.

vertical_profile_height: specifies the vertical height of the part's profile from the base boundary to where the bevel begins.

4.3.2 Boundary_curve_representation   

A Boundary_curve_representation is a set of curves that define the sheet edge. These curves define a closed area but are not necessarily restrained topologically. The curves that encompass the other curves are the outer boundary, while curves that define a closed area within the outer boundary are cutouts.

EXPRESS specification:

*)
ENTITY Boundary_curve_representation;
  bounds : SET[1:?] OF Curve;
END_ENTITY;
(*

Attribute definitions:

bounds: a set of Curve objects that defines the boundary aspect.

4.3.3 Composite_sheet_representation   

A Composite_sheet_representation is a type of Constituent_shape_representation that defines a sheet product. A sheet product is a product whose thickness is constant and small in comparison to its length and width. Most sheet products will conform to the surface on which they are placed. The shape of a sheet product can be represented in a 3-dimensional model of the actual shape, as a shape projected onto the layup surface, or as a shape projected into a 2-dimensional plane.

EXAMPLE    A typical composite sheet is 0.1 to 0.4 millimeters thick while its length and width are much greater than 25 millimeters.

A Composite_sheet_representation may be one of the following:
  • Face_based_sheet_representation,
  • Geometric_sheet_representation.

EXPRESS specification:

*)
ENTITY Composite_sheet_representation
  SUPERTYPE OF (ONEOF (Face_based_sheet_representation,
                       Geometric_sheet_representation))
  SUBTYPE OF (Constituent_shape_representation);
END_ENTITY;
(*

4.3.4 Constituent_shape_representation   

A Constituent_shape_representation is a type of Geometric_model. that defines the shape of composite part constituents. A Constituent_shape_representation is one of the following:
  • Three_d_geometry_set,
  • Composite_sheet_representation,
  • Beveled_sheet_representation.

EXPRESS specification:

*)
ENTITY Constituent_shape_representation
  ABSTRACT SUPERTYPE OF (ONEOF (Three_d_geometry_set,
                                Composite_sheet_representation,
                                Beveled_sheet_representation))
  SUBTYPE OF (Geometric_model);
END_ENTITY;
(*

4.3.5 Face_based_sheet_representation   

A Face_based_sheet_representation is a type of Composite_sheet_representation that is defined topologically.

EXPRESS specification:

*)
ENTITY Face_based_sheet_representation
  SUBTYPE OF (Composite_sheet_representation);
  face_surface : Surface;
  inner_bounds : OPTIONAL SET[1:?] OF Loop;
  outer_bound : Loop;
END_ENTITY;
(*

Attribute definitions:

face_surface: the basis surface for the face.

inner_bounds: a set of topologically defined loops that define any cutouts in the sheet. The value of this attribute need not be specified.

outer_bound: the topologically defined loop that is the outer edge of the sheet.

4.3.6 Flat_pattern_ply_shape   

A Flat_pattern_ply_shape is a type of Ply_shape It defines the edge boundary of a ply in a plane. When the shape is laid up in the assembly, it will have the proper size to define the desired ply shape.

EXPRESS specification:

*)
ENTITY Flat_pattern_ply_shape
  SUBTYPE OF (Ply_shape);
  wrapup_origin_on_flat_pattern : Cartesian_point;
  wrapup_origin_on_surface : Cartesian_point;
END_ENTITY;
(*

Attribute definitions:

wrapup_origin_on_flat_pattern: the point of the flat pattern plane that corresponds directly to the wrapup_origin_on_surface.

wrapup_origin_on_surface: the point on the basis surface that corresponds directly to the wrapup_origin_on_flat_pattern.

4.3.7 Geometric_sheet_representation   

A Geometric_sheet_representation is a type of Composite_sheet_representation that is defined by a surface and a set of curves. These curves are related in the application to that surface. The curves define a closed area but are not necessarily restrained topologically.

EXPRESS specification:

*)
ENTITY Geometric_sheet_representation
  SUBTYPE OF (Composite_sheet_representation);
  basis_surface : Surface;
  cutouts : OPTIONAL SET[1:?] OF Boundary_curve_representation;
  outer_edge : Boundary_curve_representation;
END_ENTITY;
(*

Attribute definitions:

basis_surface: the Surface used to define shape and having some known context to the part being represented. The surface may be the layup surface, inner-mold-line surface, outer-mold-line surface, or other.

cutouts: the cutouts specifies the sets of curves that define the areas to be removed from the interior of the part. The value of this attribute need not be specified.

outer_edge: the set of curves that define the outside edge (or boundary) of the part.

4.3.8 Laid_ply_shape   

A Laid_ply_shape is a type of Ply_shape that defines the shape of the ply in the actual Ply_laminate. This is the physical shape that will stair up and down over the build-up of previously laid plies.

EXPRESS specification:

*)
ENTITY Laid_ply_shape
  SUBTYPE OF (Ply_shape);
END_ENTITY;
(*

4.3.9 Loop   

A Loop is a type of Detailed_topological_model_element.

EXPRESS specification:

*)
ENTITY Loop
  SUBTYPE OF (Detailed_topological_model_element);
END_ENTITY;
(*

4.3.10 Ply_shape   

A Ply_shape is the association of a geometric representation to define the shape of a Ply.

NOTE    Each type of Ply_shape can be generated from, or related to, another Ply_shape.

EXAMPLE    A Flat_pattern_ply_shape can be generated from a Laid_ply_shape; and a View_ply_shape; can be projected to the surface and create a Surface_ply_shape.

A Ply_shape may be one of the following:
  • Projected_ply_shape,
  • Laid_ply_shape,
  • Flat_pattern_ply_shape.

EXPRESS specification:

*)
ENTITY Ply_shape
  SUPERTYPE OF (ONEOF (Projected_ply_shape,
                       Laid_ply_shape,
                       Flat_pattern_ply_shape));
  defining_model : Composite_sheet_representation;
  basis : OPTIONAL Ply_shape;
END_ENTITY;
(*

Attribute definitions:

defining_model: the geometric representation used for the Ply_shape.

basis: the shape representation that the Ply_shape is derived from or that is used as its basis of definition. The value of this attribute need not be specified.

4.3.11 Projected_ply_shape   

A Projected_ply_shape is a type of Ply_shape that has its shape modeled by a projection on a surface or a plane. The surface has some defined context for the Ply.

NOTE    A Projected_ply_shape is some projection of the actual ply. Projected shapes do not represent the build-up of the surface but all lie on the same surface or plane. These shapes are defined during ply layout in design.

A Projected_ply_shape may be one of the following:
  • View_ply_shape,
  • Surface_ply_shape.

EXPRESS specification:

*)
ENTITY Projected_ply_shape
  SUPERTYPE OF (ONEOF (View_ply_shape,
                       Surface_ply_shape))
  SUBTYPE OF (Ply_shape);
  method : projection_method;
  projection_direction : OPTIONAL Direction;
END_ENTITY;
(*

Attribute definitions:

method: identifies the reference direction used in projecting the shape on the surface. The value of method shall be one of the following:

  • projection_reference;
  • surface_normal.

projection_direction: the direction from the given representation to the actual Laid_ply_shape. The value of this attribute need not be specified.

4.3.12 Surface_ply_shape   

A Surface_ply_shape is a type of Projected_ply_shape that is defined on a surface defining some aspect of a composite part. The surface may be the layup surface, the outer mold line (OML), or the inner mold line (IML).

EXPRESS specification:

*)
ENTITY Surface_ply_shape
  SUBTYPE OF (Projected_ply_shape);
  offset : Length_data_element;
  surface_role : STRING;
END_ENTITY;
(*

Attribute definitions:

offset: the distance from the modeled surface to the layup surface of the part. The offset distance would be zero if the modeled surface is the layup surface.

surface_role: the context of the surface defined to the actual part and should be limited to text defining contexts such as 'layup surface', 'outer mold line', and 'inner mold line'.

4.3.13 Three_d_geometry_set   

A Three_d_geometry_set is a type of Constituent_shape_representation that defines 3D boundary curve and surface geometry without topology. The surfaces used in the representation have a defined model surface context.

EXPRESS specification:

*)
ENTITY Three_d_geometry_set
  SUBTYPE OF (Constituent_shape_representation);
  basis_role : STRING;
  basis_surface : Surface;
  defining_boundary : SET[1:?] OF Curve;
END_ENTITY;
(*

Attribute definitions:

basis_role: the context of the basis surface for the part. 'layup surface', 'outer mold line', and 'inner mold line' are specific contexts.

basis_surface: the Surface used to define the part shape. The context of the surface is defined in the attribute basis_role.

defining_boundary: the set of Curve objects that define the edge aspect of a part.

4.3.14 View_ply_shape   

A View_ply_shape is a type of Projected_ply_shape in which the projection is to a plane such as a drawing view. The projection direction establishes the relationship of the plane to the actual part.

EXPRESS specification:

*)
ENTITY View_ply_shape
  SUBTYPE OF (Projected_ply_shape);
END_ENTITY;
(*
