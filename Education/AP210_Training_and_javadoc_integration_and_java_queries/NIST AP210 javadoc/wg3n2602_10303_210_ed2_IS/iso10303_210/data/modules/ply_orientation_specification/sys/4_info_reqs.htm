4 Information requirements

This clause specifies the information requirements for the Ply orientation specification application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Ply_orientation_specification_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Ply_orientation_specification_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Part_and_zone_laminate_tables_arm;    --  ISO/TS 10303-1770
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Part_and_zone_laminate_tables_arm ISO/TS 10303-1770

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 angle_select   

The angle_select type allows for the designation of the data types Axis_placement_3d, Curve, Direction and Point_path.

EXPRESS specification:

*)
TYPE angle_select = SELECT
   (Axis_placement_3d,
    Curve,
    Direction,
    Point_path);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Direction_of_fiber_orientation_within_a_ply   

A Direction_of_fiber_orientation_within_a_ply is a type of Representation.

EXPRESS specification:

*)
ENTITY Direction_of_fiber_orientation_within_a_ply
  SUBTYPE OF (Representation);
  SELF\Representation.context_of_items : Geometric_coordinate_space;
  SELF\Representation.items : SET[1:?] OF angle_select;
INVERSE
  orientations : SET[1:1] OF Ply_orientation_angle FOR rep_1;
END_ENTITY;
(*

Attribute definitions:

context_of_items: an attribute inherited from the Representation shall be redeclared as the Geometric_coordinate_space for the Direction_of_fiber_orientation_within_a_ply.

items: an attribute inherited from the Representation shall be redeclared as the angle_select for the Direction_of_fiber_orientation_within_a_ply. There shall exist exactly one angle_select for the Direction_of_fiber_orientation_within_a_ply.

orientations: specifies an inverse relationship that specifies that the existence of the Direction_of_fiber_orientation_within_a_ply is dependent on the existence of the Ply_orientation_angle that specifies the Direction_of_fiber_orientation_within_a_ply as its rep_1. There shall exist exactly one Ply_orientation_angle for the Direction_of_fiber_orientation_within_a_ply.

4.3.2 Draped_orientation_angle   

A Draped_orientation_angle is a type of Ply_orientation_angle.

EXPRESS specification:

*)
ENTITY Draped_orientation_angle
  SUBTYPE OF (Ply_orientation_angle);
END_ENTITY;
(*

4.3.3 Laid_orientation_angle   

A Laid_orientation_angle is a type of Ply_orientation_angle.

EXPRESS specification:

*)
ENTITY Laid_orientation_angle
  SUBTYPE OF (Ply_orientation_angle);
END_ENTITY;
(*

4.3.4 Ply_orientation_angle   

A Ply_orientation_angle is a type of Representation_relationship. A Ply_orientation_angle may be one either a Draped_orientation_angle or a Laid_orientation_angle.

EXPRESS specification:

*)
ENTITY Ply_orientation_angle
  SUPERTYPE OF (ONEOF (Draped_orientation_angle,
                       Laid_orientation_angle))
  SUBTYPE OF (Representation_relationship);
  angle_reference : angle_select;
  SELF\Representation_relationship.rep_1 : Direction_of_fiber_orientation_within_a_ply;
  SELF\Representation_relationship.rep_2 : Reinforcement_orientation_basis;
END_ENTITY;
(*

Attribute definitions:

angle_reference: specifies a role of the angle_select for the Ply_orientation_angle.

rep_1: an attribute inherited from the Representation_relationship shall be redeclared as the Direction_of_fiber_orientation_within_a_ply for the Ply_orientation_angle.

rep_2: an attribute inherited from the Representation_relationship shall be redeclared as the Reinforcement_orientation_basis for the Ply_orientation_angle.

4.3.5 Point_and_vector   

A Point_and_vector is a type of Detailed_geometric_model_element. A Point_and_vector is a position in space combined with two directions.

EXPRESS specification:

*)
ENTITY Point_and_vector
  SUBTYPE OF (Detailed_geometric_model_element);
  location : point_select;
  vector : LIST[1:2] OF Direction;
END_ENTITY;
(*

Attribute definitions:

location: specifies a role of the point_select for the Point_and_vector.

vector: specifies a role of the Direction for the Point_and_vector. There shall exist one or two Directions for the Point_and_vector.

4.3.6 Point_path   

A Point_path is a type of Detailed_geometric_model_element. A Point_path is a list of points and directions specifying the fiber orientations for a Ply.

EXPRESS specification:

*)
ENTITY Point_path
  SUBTYPE OF (Detailed_geometric_model_element);
  directions : LIST[1:?] OF Point_and_vector;
END_ENTITY;
(*

Attribute definitions:

directions: specifies a role of the Point_and_vector for the Point_path.directions. There shall exist one or more Point_and_vectors for the Point_path.directions.

4.3.7 Reinforcement_orientation_basis   

A Reinforcement_orientation_basis is a type of Representation. A Reinforcement_orientation_basis is the definition of direction defined in terms of a Ply_laminate or Composite_assembly. It is the basis for interpretation of the fiber orientation within a ply.

EXPRESS specification:

*)
ENTITY Reinforcement_orientation_basis
  SUBTYPE OF (Representation);
  SELF\Representation.context_of_items : Geometric_coordinate_space;
  SELF\Representation.items : SET[1:1] OF Axis_placement_3d;
DERIVE
  basis_location : Cartesian_point := SELF\Representation.items[1]\Axis_placement.origin;
  orientation : Direction := SELF\Representation.items[1]\Axis_placement.x_axis;
INVERSE
  tables : SET[1:?] OF Laminate_table FOR basis;
  orientations : SET[1:?] OF Ply_orientation_angle FOR rep_2;
END_ENTITY;
(*

Attribute definitions:

context_of_items: an attribute inherited from the Representation shall be redeclared as the Geometric_coordinate_space for the Reinforcement_orientation_basis.

items: an attribute inherited from the Representation shall be redeclared as the Axis_placement_3d for the Reinforcement_orientation_basis. There shall exist exactly one Axis_placement_3d for the Reinforcement_orientation_basis.

basis_location: specifies a role of the point_select for the Reinforcement_orientation_basis.

orientation: specifies a role of the Direction for the Reinforcement_orientation_basis.

tables: specifies an inverse relationship that specifies that the existence of the Reinforcement_orientation_basis is dependent on the existence of the Laminate_table that specifies the Reinforcement_orientation_basis as its basis. There shall be one or more Laminate_tables for a Reinforcement_orientation_basis.

orientations: specifies an inverse relationship that specifies that the existence of the Reinforcement_orientation_basis is dependent on the existence of the Ply_orientation_angle that specifies the Reinforcement_orientation_basis as its rep_2. There shall be one or more Ply_orientation_angles for a Reinforcement_orientation_basis.

4.4 ARM subtype constraint definition

This subclause specifies the ARM subtype constraint for this module. The subtype constraint places a constraint on the possible super-type / subtype instantiations. The ARM subtype constraint and definition is specified below.

4.4.1 pos_detailed_geometric_model_element_subtypes   

The pos_detailed_geometric_model_element_subtypes constraint specifies a constraint that applies to instances of subtypes of Detailed_geometric_model_element.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT pos_detailed_geometric_model_element_subtypes FOR Detailed_geometric_model_element;
  ONEOF (Point_and_vector,
         Point_path);
END_SUBTYPE_CONSTRAINT;
