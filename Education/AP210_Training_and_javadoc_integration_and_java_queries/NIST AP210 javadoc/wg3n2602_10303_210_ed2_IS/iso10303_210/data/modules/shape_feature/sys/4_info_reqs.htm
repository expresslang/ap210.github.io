4 Information requirements

This clause specifies the information requirements for the Shape feature application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Shape_feature_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Shape_feature_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Contextual_shape_positioning_arm;    --  ISO/TS 10303-1027

USE FROM Property_assignment_arm;    --  ISO/TS 10303-1030

REFERENCE FROM Characterizable_object_arm    --  ISO/TS 10303-1765
  (bag_to_set);
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Contextual_shape_positioning_arm ISO/TS 10303-1027
Property_assignment_arm ISO/TS 10303-1030
Characterizable_object_arm ISO/TS 10303-1765

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 sf_property_assignment_select   

The sf_property_assignment_select type is an extension of the property_assignment_select type. It adds the data types Characterizable_object and Shape_element to the list of alternate data types.

EXPRESS specification:

*)
TYPE sf_property_assignment_select = SELECT BASED_ON property_assignment_select WITH
   (Characterizable_object,
    Shape_element);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Instanced_feature   

An Instanced_feature is a type of Shape_feature_occurrence and a type of Shape_feature_definition. An Instanced_feature is the identification of a feature as an element of the Item_shape of a product. All geometric data defining the feature are defined in the coordinate space of the product.

EXPRESS specification:

*)
ENTITY Instanced_feature
  SUBTYPE OF (Shape_feature_definition, Shape_feature_occurrence);
END_ENTITY;
(*

4.3.2 Placed_feature   

A Placed_feature is a type of Shape_feature_occurrence. A Placed_feature is the occurrence of a feature on an Item_shape of a product where the definition of the feature is given in its own coordinate space.

EXPRESS specification:

*)
ENTITY Placed_feature
  SUBTYPE OF (Shape_feature_occurrence);
  definition : Shape_feature_definition;
END_ENTITY;
(*

Attribute definitions:

definition: specifies a role of the Shape_feature_definition for the Placed_feature.

4.3.3 Shape_feature_definition   

A Shape_feature_definition is a type of Characterizable_object. A Shape_feature_definition is a set of parameters and type information representing a feature that is either defined as a component of an item (Instanced_feature) or independently in its own coordinate space for multiple use (Placed_feature).

EXPRESS specification:

*)
ENTITY Shape_feature_definition
  SUBTYPE OF (Characterizable_object);
DERIVE
  feature_model : SET[1:?] OF Geometric_model := get_geometric_model(of_shape);
INVERSE
  of_shape : Item_shape FOR described_element;
WHERE
  WR1: EXISTS(feature_model);
END_ENTITY;
(*

Attribute definitions:

feature_model: specifies a role of the Geometric_model for the Shape_feature_definition.

of_shape: specifies an inverse relationship that specifies that the existence of the Shape_feature_definition is dependent on the existence of the Item_shape that specifies the Shape_feature_definition as its described_element.

Formal propositions:

WR1: The feature_model shall exist.

4.3.4 Shape_feature_occurrence   

A Shape_feature_occurrence is a type of Shape_element. A Shape_feature_occurrence is one of an Instanced_feature or a Placed_feature.

EXPRESS specification:

*)
ENTITY Shape_feature_occurrence
  ABSTRACT SUPERTYPE OF (ONEOF (Instanced_feature,
                                Placed_feature))
  SUBTYPE OF (Shape_element);
END_ENTITY;
(*

4.4 ARM function definition

This subclause specifies the ARM function for this module. The ARM function and definition is specified below.

4.4.1 get_geometric_model

The get_geometric_model function returns the set of Geometric_models that are associated to input by means of Shape_description_associations.

EXPRESS specification:

*)
FUNCTION get_geometric_model (input : Item_shape) :SET[0:?] OF Geometric_model; 
LOCAL
  sda : SET OF Shape_description_association := [];
  gm : SET OF Geometric_model := [];
 END_LOCAL;
 
  sda := bag_to_set(USEDIN(input,
	 	'SHAPE_PROPERTY_ASSIGNMENT_ARM.' + 
	 	'SHAPE_DESCRIPTION_ASSOCIATION.' + 'REPRESENTED_CHARACTERISTIC'));
 
 REPEAT i := 1 to SIZEOF(sda) BY 1;
  gm := gm + sda[i].representation;
 END_REPEAT;
 RETURN(gm);
END_FUNCTION;
(*

Argument definitions:

input: the specified set of Item_shape.



*)
END_SCHEMA;  -- Shape_feature_arm
