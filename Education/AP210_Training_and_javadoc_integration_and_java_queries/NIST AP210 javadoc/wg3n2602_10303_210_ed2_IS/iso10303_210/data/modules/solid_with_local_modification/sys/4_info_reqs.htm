4 Information requirements

This clause specifies the information requirements for the Solid with local modification application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Solid_with_local_modification_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Solid_with_local_modification_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Basic_geometric_topology_arm;    --  ISO/TS 10303-1323

USE FROM Procedural_solid_model_arm;    --  ISO/TS 10303-1318

USE FROM Topologically_bounded_surface_arm;    --  ISO/TS 10303-1511
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Basic_geometric_topology_arm ISO/TS 10303-1323
Procedural_solid_model_arm ISO/TS 10303-1318
Topologically_bounded_surface_arm ISO/TS 10303-1511

NOTE 2   See Annex C, Figures C.1, C.2, C.3, C.4and C.5 for a graphical representation of this schema.

4.2 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.2.1 blend_end_condition   

A blend_end_condition is a select type used to specify the termination conditions when blending a continuous path of edges in a solid. type of end condition required when blending a Path_modified_solid.

EXPRESS specification:

*)
TYPE blend_end_condition = SELECT
   (Edge_curve,
    Point_on_curve,
    Vertex);
END_TYPE;
(*

4.2.2 blend_radius_variation   

A blend_radius_variation is an enumeration of the types of function controlling the variation of radius when blending a continuous path of edges in a solid.

EXPRESS specification:

*)
TYPE blend_radius_variation = ENUMERATION OF
   (linear,
    cubic,
    unspecified);
END_TYPE;
(*

Enumerated item definitions:

linear: the blend radius varies linearly between radius definition points;

cubic: the blend radius varies as a cubic function between radius definition points;

unspecified: the blend radius variation function is not specified.

4.2.3 sculpting_surface   

A sculpting_surface is a select type used to specify the type of surface used to cut a solid model to create a smaller solid.

EXPRESS specification:

*)
TYPE sculpting_surface = SELECT
   (Advanced_face,
    Advanced_open_shell,
    Surface);
END_TYPE;
(*

4.2.4 solid_with_placed_feature   

A solid_with_placed_feature is a select type used to specify the type of placed feature which may be used to create a pattern.

EXPRESS specification:

*)
TYPE solid_with_placed_feature = SELECT
   (Solid_with_circular_pattern,
    Solid_with_depression,
    Solid_with_protrusion,
    Solid_with_rectangular_pattern);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Advanced_open_shell   

An Advanced_open_shell is a type of Detailed_topological_model_element that is a collection of faces forming an open shell.

EXPRESS specification:

*)
ENTITY Advanced_open_shell
  SUBTYPE OF (Detailed_topological_model_element);
  shell_faces : SET[1:?] OF Advanced_face;
END_ENTITY;
(*

Attribute definitions:

shell_faces: the set of faces defining the geometry and topology of the Advanced_open_shell

4.3.2 Complex_deleted_face_solid   

A Complex_deleted_face_solid is a type of Deleted_face_solid in which selected faces have specified thicknesses differing from the general thickness inherited from the supertype Deleted_face_solid.

EXPRESS specification:

*)
ENTITY Complex_deleted_face_solid
  SUBTYPE OF (Deleted_face_solid);
  thickened_face_list : LIST[1:?] OF SET[1:?] OF Advanced_face;
  thickness_list : LIST[1:?] OF length_measure;
WHERE
  WR1: SIZEOF(thickened_face_list) = SIZEOF(thickness_list);
END_ENTITY;
(*

Attribute definitions:

thickened_face_list: the list of sets of faces having different thickness values.

thickness_list: the list of individual thickness values to be assigned to the faces in thickened_face_list

Formal propositions:

WR1: The number of sets of faces in thickened_face_list shall be equal to the size of thickness_list.

4.3.3 Conical_stepped_hole_transition   

A Conical_stepped_hole_transition is a type of Detailed_geometric_model_element is a conical face interposed at a junction between two cylindrical segments of a Solid_with_stepped_round_hole. The cone is characterized in terms of its apex angle and its base radius. The apex of the cone always lies on the hole axis, on that side of the junction for which the hole radius is smaller, and the base radius of the cone is defined on the planar surface normal to that axis and intersecting it at the point where the radius change occurs. The apex angle is positive when the transition is from a larger to a smaller hole radius, in the sense of increasing hole depth, and negative if it is from a smaller to a larger radius.

EXPRESS specification:

*)
ENTITY Conical_stepped_hole_transition
  SUBTYPE OF (Detailed_geometric_model_element);
  apex_angle : plane_angle_measure;
  cone_base_radius : length_measure;
  transition_number : INTEGER;
WHERE
  WR1: transition_number > 0;
  WR2: cone_base_radius > 0.0;
END_ENTITY;
(*

Attribute definitions:

apex_angle: the apex angle of the conical surface defining the transition.

cone_base_radius: the base radius of the conical transition surface.

transition_number: the number of the transition defined by the conical shape.

Formal propositions:

WR1: The transition_number shall be positive.

WR2: The cone_base_radius shall be positive.

4.3.4 Deleted_face_solid   

A Deleted_face_solid is a type of Modified_solid in which some of the faces are removed and the remaining faces are thickened to produce a solid in the form of a thick shell. This operation is sometimes called `shelling', or `hollowing'. A Deleted_face_solid may be a Deleted_face_solid_with_offsets or a Complex_deleted_face_solid.

EXPRESS specification:

*)
ENTITY Deleted_face_solid
  SUPERTYPE OF (ONEOF (Deleted_face_solid_with_offsets,
                       Complex_deleted_face_solid))
  SUBTYPE OF (Modified_solid);
  deleted_faces : SET[1:?] OF Advanced_face;
  thickness : length_measure;
WHERE
  WR1: thickness <> 0.0;
END_ENTITY;
(*

Attribute definitions:

deleted_faces: the set of faces which are to be removed from the solid.

thickness: the thickness of the remaining faces of the modified solid. A positive value indicates that material is to be added in the direction of the face normal, if thickness is negative the added material is below the face.

Formal propositions:

WR1: The thickness shall be non-zero.

4.3.5 Deleted_face_solid_with_offsets   

A Deleted_face_solid_with_offsets is a type of Deleted_face_solid for which two thickness offset values are specified to define the thickening of the remaining faces.

EXPRESS specification:

*)
ENTITY Deleted_face_solid_with_offsets
  SUBTYPE OF (Deleted_face_solid);
  thickness2 : length_measure;
WHERE
  WR1: thickness2 <> 0.0;
  WR2: SELF\Deleted_face_solid.thickness <> thickness2;
END_ENTITY;
(*

Attribute definitions:

thickness2: the second thickness value to be applied to the retained faces of the base solid. The retained faces will have a thickness equal to the difference between thickness and thickness2.

Formal propositions:

WR1: The thickness2 shall be non-zero.

WR2: The thickness2 shall not be equal to thickness

4.3.6 Path_modified_solid   

A Path_modified_solid is a type of Solid_with_edge_modification in which the edges to be modified form a continuous path.

NOTE    This entity is incompletely modelled in the ARM, the corresponding MIM entity contains a constraint to ensure that the specified edges have the correct configuration.

EXPRESS specification:

*)
ENTITY Path_modified_solid
  SUBTYPE OF (Solid_with_edge_modification);
END_ENTITY;
(*

4.3.7 Path_modified_solid_with_end_conditions   

A Path_modified_solid_with_end_conditions is a type of Path_modified_solid for which the blend is terminated at points other than the start and end vertices of the path of selected edges.

NOTE    This entity is incompletely modelled in the ARM, the corresponding MIM entity defines the types of termination condition permitted and contains constraints to ensure consistency.

EXPRESS specification:

*)
ENTITY Path_modified_solid_with_end_conditions
  SUBTYPE OF (Path_modified_solid);
  end_conditions : LIST[2:2] OF blend_end_condition;
END_ENTITY;
(*

Attribute definitions:

end_conditions: the list of 2 end_conditions which define respectively the start and end of the edge blending process.

4.3.8 Sculptured_solid   

A Sculptured_solid is a type of Modified_solid in which a solid is cut by a surface, face or shell to remove part of the material leaving a smaller solid model.

EXPRESS specification:

*)
ENTITY Sculptured_solid
  SUBTYPE OF (Modified_solid);
  cutting_surface : sculpting_surface;
  positive_side : BOOLEAN;
END_ENTITY;
(*

Attribute definitions:

cutting_surface: the surface, face, or open shell used to divide the remaining solid from the material removed.

positive_side: an indicator of the side of the sculpturing element on which the result of the cutting operation is to be retained. If its value is true the solid on the positive side of the surface, face or shell is retained, and vice versa. The definition of the positive side of the sculpturing element depends on the nature of that element. For a surface, it is the side containing the positive surface normal vector. For a face_surface, or an Advanced_open_shell, it is the side containing the topological normal to the face or shell respectively.

4.3.9 Sculptured_solid_with_selection   

A Sculptured_solid_with_selection is a type of Sculptured_solid to be used when the simple definition of a Sculptured_solid does not give a unique result with 2 or more remaining solids.

EXPRESS specification:

*)
ENTITY Sculptured_solid_with_selection
  SUBTYPE OF (Sculptured_solid);
  retained_part : Detailed_topological_model_element;
END_ENTITY;
(*

Attribute definitions:

retained_part: an Detailed_topological_model_element which defines the boundary of the solid to be retained.

4.3.10 Solid_with_angled_chamfer   

A Solid_with_angled_chamfer is a type of Solid_with_edge_modification in which the chamfers are defined in terms of an offset curve on one of the faces adjoining at the edge concerned and an offset angle measured with respect to the surface of that face. If this face is not planar, the chamfer angle is measured with respect to the tangent plane to the underlying surface at points on the offset curve.

EXPRESS specification:

*)
ENTITY Solid_with_angled_chamfer
  SUBTYPE OF (Solid_with_edge_modification);
  offset_distance : length_measure;
  left_offset : BOOLEAN;
  offset_angle : plane_angle_measure;
WHERE
  WR1: offset_distance > 0.0;
  WR2: offset_angle > 0.0;
END_ENTITY;
(*

Attribute definitions:

offset_distance: the distance from the modified edge to the end of the chafer on the selected adjacent face.

left_offset: a BOOLEAN which indicates whether or not the reference face, used to define the angle of the chamfer, is to the left of the chamfered edge.

offset_angle: the angle between the chamfer and the selected face.

Formal propositions:

WR1: The offset_distance shall be positive.

WR2: The offset_angle shall be positive.

4.3.11 Solid_with_circular_pattern   

A Solid_with_circular_pattern is a type of Modified_solid that has a regular circular pattern replicating a previously defined shape modification feature.

EXPRESS specification:

*)
ENTITY Solid_with_circular_pattern
  SUBTYPE OF (Modified_solid);
  location : Axis_placement;
  replicated_element : solid_with_placed_feature;
  replicate_count : INTEGER;
  angular_spacing : plane_angle_measure;
  radial_alignment : BOOLEAN;
  reference_point : Cartesian_point;
WHERE
  WR1: replicate_count > 0;
END_ENTITY;
(*

Attribute definitions:

location: the position and orientation of the circular pattern of shape modifications.

replicated_element: the shape modification feature that is replicated to form the circular pattern, this may be a depression, a protrusion or another pattern.

replicate_count: the number of repetitions of the replicated element, excluding the original.

angular_spacing: the angular displacement between successive replicates.

radial_alignment: a BOOLEAN indicating whether, or not, the replicates are individually rotated about the centre of rotation. If false, each point in the pattern has the same translation as the reference point, so that the orientation of the original configuration is maintained.

reference_point: a point associated with the original configuration to be replicated, whose projected distance from the axis of rotation determines the radius of the circular pattern. The replicates have corresponding transformed reference points at equal angular spacings about the axis of the circular pattern.

Formal propositions:

WR1: The replicate_count shall be of positive.

4.3.12 Solid_with_circular_pocket   

A Solid_with_circular_pocket is a type of Solid_with_pocket in which the depression is in the form of a circular pocket.

EXPRESS specification:

*)
ENTITY Solid_with_circular_pocket
  SUBTYPE OF (Solid_with_pocket);
  pocket_radius : length_measure;
WHERE
  WR1: SELF\Solid_with_pocket.floor_blend_radius <= pocket_radius;
  WR2: pocket_radius > 0.0;
END_ENTITY;
(*

Attribute definitions:

pocket_radius: the radius of the cylinder forming the pocket.

Formal propositions:

WR1: The floor_blend_radius shall be less than the radius of the cylindrical hole.

WR2: The pocket_radius shall be positive.

4.3.13 Solid_with_circular_protrusion   

A Solid_with_circular_protrusion is a type of Solid_with_protrusion such that the protrusion has a circular cross-section.

EXPRESS specification:

*)
ENTITY Solid_with_circular_protrusion
  SUBTYPE OF (Solid_with_protrusion);
  radius : length_measure;
WHERE
  WR1: radius >= 0.0;
END_ENTITY;
(*

Attribute definitions:

radius: the radius of the circular cross-section of the protrusion.

Formal propositions:

WR1: The radius shall be of positive.

4.3.14 Solid_with_conical_bottom_hole   

A Solid_with_conical_bottom_hole is a type of Solid_with_stepped_round_hole having a conical bottom.

EXPRESS specification:

*)
ENTITY Solid_with_conical_bottom_hole
  SUBTYPE OF (Solid_with_stepped_round_hole);
  semi_apex_angle : plane_angle_measure;
  tip_radius : length_measure;
WHERE
  WR1: tip_radius < SELF\Solid_with_stepped_round_hole.segment_radii[segments];
  WR2: tip_radius >= 0.0;
  WR3: semi_apex_angle > 0.0;
END_ENTITY;
(*

Attribute definitions:

semi_apex_angle: the semi-angle at the apex of the conical surface at the bottom of the hole.

tip_radius: radius of the spherical surface rounding off the tip of the conical surface at the bottom of the hole.

Formal propositions:

WR1: The tip_radius shall be less than the radius of the final cylindrical segment of the hole.

WR2: The tip_radius shall not be negative but may be zero.

WR3: The semi_apex_angle shall be positive.

4.3.15 Solid_with_curved_slot   

A Solid_with_curved_slot is a type of Solid_with_slot on which a slot has been created by sweeping the slot profile along a curve.

EXPRESS specification:

*)
ENTITY Solid_with_curved_slot
  SUBTYPE OF (Solid_with_slot);
  slot_centreline : Detailed_geometric_model_element;
END_ENTITY;
(*

Attribute definitions:

slot_centreline: the curve defining the centre line of the slot.

NOTE    This attribute is incompletely specified in the ARM model, it is fully defined in the MIM where it is mapped to a bounded_curve instance.

4.3.16 Solid_with_depression   

A Solid_with_depression is a type of Modified_solid that has depressions in one or more faces. The particular types of depression are modelled in the subtypes. A Solid_with_depression is a Solid_with_through_depression and/or a Solid_with_stepped_round_hole or a Solid_with_pocket or a Solid_with_slot or a Solid_with_groove.

EXPRESS specification:

*)
ENTITY Solid_with_depression
  ABSTRACT SUPERTYPE OF (Solid_with_through_depression
                         ANDOR ONEOF (Solid_with_stepped_round_hole,
                              Solid_with_pocket,
                              Solid_with_slot,
                              Solid_with_groove))
  SUBTYPE OF (Modified_solid);
  location : Axis_placement;
  depth : length_measure;
WHERE
  WR1: depth > 0.0;
END_ENTITY;
(*

Attribute definitions:

location: the position and orientation of the depression modifying the shape of the solid.

depth: the depth of the depression.

Formal propositions:

WR1: The depth shall be positive.

4.3.17 Solid_with_double_offset_chamfer   

A Solid_with_double_offset_chamfer is a type of Solid_with_edge_modification in which the modified edges are chamfered connecting points at different distances from the modified edge on adjacent faces.

EXPRESS specification:

*)
ENTITY Solid_with_double_offset_chamfer
  SUBTYPE OF (Solid_with_edge_modification);
  left_offset_distance : length_measure;
  right_offset_distance : length_measure;
WHERE
  WR1: left_offset_distance > 0.0;
  WR2: right_offset_distance > 0.0;
END_ENTITY;
(*

Attribute definitions:

left_offset_distance: the distance from the modified edge to the end of the chafer on face to the left of the edge.

right_offset_distance: the distance from the modified edge to the end of the chafer on face to the right of the edge.

Formal propositions:

WR1: The left_offset_distance shall be positive.

WR2: The right_offset_distance shall be positive.

4.3.18 Solid_with_edge_modification   

A Solid_with_edge_modification is a type of Modified_solid that specifies the edges to be modified and the type of modification required. The subtypes of this entity define in detail the types of possible edge modification. A Solid_with_edge_modification may be a Path_modified_solid and/or one of a Solid_with_fixed_radius_edge_blend, a Solid_with_simple_chamfer, a Solid_with_double_offset_chamfer, a Solid_with_angled_chamfer.

EXPRESS specification:

*)
ENTITY Solid_with_edge_modification

                ANDOR ONEOF (Solid_with_fixed_radius_edge_blend,

                     Solid_with_double_offset_chamfer,
                     Solid_with_angled_chamfer))
  SUBTYPE OF (Modified_solid);
  blended_edges : LIST[1:?] OF Edge;
END_ENTITY;
(*

Attribute definitions:

blended_edges: the list of edges to be modified.

4.3.19 Solid_with_fixed_radius_edge_blend   

A Solid_with_fixed_radius_edge_blend is a type of Solid_with_edge_modification in which the modified edges are blended in the form of a rolling ball blend of constant radius.

EXPRESS specification:

*)
ENTITY Solid_with_fixed_radius_edge_blend

  radius : length_measure;
WHERE
  WR1: radius > 0.0;
END_ENTITY;
(*

Attribute definitions:

radius: the radius of the rounded edges created by the blending process.

Formal propositions:

WR1: The radius shall be positive.

4.3.20 Solid_with_flat_bottom_hole   

A Solid_with_flat_bottom_hole is a type of Solid_with_stepped_round_hole having a flat bottom.

EXPRESS specification:

*)
ENTITY Solid_with_flat_bottom_hole
  SUBTYPE OF (Solid_with_stepped_round_hole);

fillet_radius: the radius of the fillet between the flat bottom of the hole and its cylindrical shaft.

4.3.22 Solid_with_general_protrusion   

A Solid_with_general_protrusion is a type of Solid_with_protrusion such that the protrusion is a general planar area.

reference_point: a point defining the location of the profile.

4.3.23 Solid_with_groove   

A Solid_with_groove is a type of Solid_with_depression that results from the creation of a depression extending around a surface of revolution in such a way that its cross-sections are constant in any plane containing the axis of that surface. The cross-section is of symmetric trapezoidal form, which includes as special cases rectangular sections, sections with draft angles and V-sections.

EXPRESS specification:

*)
ENTITY Solid_with_groove
  SUBTYPE OF (Solid_with_depression);
  groove_radius : length_measure;
  groove_width : length_measure;
  draft_angle : plane_angle_measure;
  floor_fillet_radius : length_measure;
  external_groove : BOOLEAN;
WHERE
  WR1: NOT('SOLID_WITH_LOCAL_MODIFICATION_ARM.SOLID_WITH_THROUGH_DEPRESSION' IN TYPEOF(SELF));
  WR2: groove_radius > 0.0;
  WR3: groove_width > 0.0;
  WR4: floor_fillet_radius >= 0.0;
END_ENTITY;
(*

Attribute definitions:

groove_radius: the radius of the cylindrical floor surface of the groove.

groove_width: the width of the groove.

draft_angle: the draft angle of the groove.

floor_fillet_radius: the radius of the fillet at the corners of the floor of the groove.

external_groove: a BOOLEAN value which indicates whether the groove is on an external surface of the solid (TRUE), or internal surface (FALSE).

Formal propositions:

WR1: The Solid_with_groove shall not be of type Solid_with_through_depression

WR2: The groove_radius shall be positive.

WR3: The groove_width shall be positive.

WR4: The floor_fillet_radius shall be positive or zero.

4.3.24 Solid_with_incomplete_circular_pattern   

A Solid_with_incomplete_circular_pattern is a type of Solid_with_circular_pattern in which some of the replicates have been omitted.

EXPRESS specification:

*)
ENTITY Solid_with_incomplete_circular_pattern
  SUBTYPE OF (Solid_with_circular_pattern);
  omitted_instances : SET[1:?] OF INTEGER;
WHERE
  WR1: SIZEOF(omitted_instances) < SELF\Solid_with_circular_pattern.replicate_count;
  WR2: SIZEOF(QUERY(q <* omitted_instances | q > SELF\Solid_with_circular_pattern.replicate_count)) = 0;
END_ENTITY;
(*

Attribute definitions:

omitted_instances: the set of integers giving the index numbers of the omitted replicates in the pattern.

Formal propositions:

WR1: The number of omitted instances shall be less than replicate_count

WR2: No integer identifier in the set of omitted replicates shall have a value that is greater than replicate_count

4.3.25 Solid_with_incomplete_rectangular_pattern   

A Solid_with_incomplete_rectangular_pattern is a type of Solid_with_rectangular_pattern in which some of the replicates have been omitted.

EXPRESS specification:

*)
ENTITY Solid_with_incomplete_rectangular_pattern
  SUBTYPE OF (Solid_with_rectangular_pattern);
  omitted_instances : SET[1:?] OF LIST[1:2] OF INTEGER;
WHERE
  WR1: NOT([1,1] IN omitted_instances);
  WR2: SIZEOF(omitted_instances) < ((SELF\Solid_with_rectangular_pattern.row_count * SELF\Solid_with_rectangular_pattern.column_count) - 1);
  WR3: SIZEOF(QUERY(q <* omitted_instances | ((q[1] > SELF\Solid_with_rectangular_pattern.row_count) OR (q[2] > SELF\Solid_with_rectangular_pattern.column_count)))) = 0;
END_ENTITY;
(*

Attribute definitions:

omitted_instances: the set of pairs of integers giving the row and column numbers of the omitted replicates in the pattern.

Formal propositions:

WR1: The original modification feature in row 1, column 1 of the pattern shall not be omitted.

WR2: The total number of omitted replicates shall be less than the number of instances in the pattern.

WR3: The row index of any omitted replicate shall not exceed the row count of the rectangular pattern, and no column index shall exceed the column count.

4.3.26 Solid_with_offset_faces   

A Solid_with_offset_faces is a type of Modified_solid in which some of the faces are offset in the direction of the face normal.

EXPRESS specification:

*)
ENTITY Solid_with_offset_faces
  SUBTYPE OF (Modified_solid);
  offset_faces : LIST[1:?] OF SET[1:?] OF Advanced_face;
  offset_distances : LIST[1:?] OF length_measure;
WHERE
  WR1: SIZEOF(offset_faces) = SIZEOF(offset_distances);
END_ENTITY;
(*

Attribute definitions:

offset_faces: the list of sets of faces which are to be modified.

offset_distances: the list of distances by which the modified faces are offset. The nth. length in this list is applied to the nth. set of faces in the offset_faces list. A positive value makes the solid locally larger, a negative value subtracts material in the vicinity of that set of faces.

Formal propositions:

WR1: The number of offset_distances shall be equal to the number of offset_faces.

4.3.27 Solid_with_pocket

4.3.33 Solid_with_slot

4.4 ARM function definition

This subclause specifies the ARM function for this module. The ARM function and definition is specified below.

4.4.1 calculate_depth

The function calculate_depth calculates the total depth of a Solid_with_stepped_round_hole by adding together the depths of the individual segments.

EXPRESS specification:

*)
FUNCTION calculate_depth (swsrh : Solid_with_stepped_round_hole) :length_measure; 
LOCAL
      i  : INTEGER;
      td : length_measure := 0;
    END_LOCAL;
    REPEAT i := 1 TO swsrh.segments;
      td := td + swsrh.segment_depths[i];
    END_REPEAT;
    RETURN(td);
END_FUNCTION;
(*

Argument definitions:

swsrh: (Input) The Solid_with_stepped_round_hole for which the total depth of the hole is to be computed.
