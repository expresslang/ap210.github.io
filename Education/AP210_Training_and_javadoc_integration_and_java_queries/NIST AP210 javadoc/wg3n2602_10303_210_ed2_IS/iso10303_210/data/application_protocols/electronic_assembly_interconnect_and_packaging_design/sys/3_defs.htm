3 Terms, definitions and abbreviations

3.1 Terms defined in ISO 10303-1

For the purposes of this document, the following terms defined in ISO 10303-1 apply:

  • application;
  • application object;
  • application activity model;
  • application interpreted model;
  • application protocol;
  • application reference model;
  • conformance class;
  • data;
  • implementation method;
  • information;
  • integrated resource;
  • PICS proforma;
  • product;
  • product data;
  • protocol implementation conformance statement.

3.2 Terms defined in ISO 10303-31

For the purposes of this document, the following terms defined in ISO 10303-31 apply:

  • conformance.

3.3 Terms defined in ISO 10303-202

For the purposes of this document, the following terms defined in ISO 10303-202 apply:

  • application interpreted construct.

3.3.1
application module

reusable collection of a scope statement, information requirements, mappings and module interpreted model that supports a specific usage of product data across multiple application contexts

3.3.2
common resources

a collection of information models, specified in EXPRESS language, that can be reused to specify application specific information models within the domain of industrial data

NOTE 1   The resource constructs defined by application modules are those defined in their MIM schema.

3.3.3
module interpreted model

information model that uses the common resources necessary to satisfy the information requirements and constraints of an application reference model, within an application module

3.4 Abbreviated terms

For the purposes of this document, the following abbreviated terms apply:

AAM application activity model
AP application protocol
ARM application reference model
MIM module interpreted model
PICS protocol implementation conformance statement
