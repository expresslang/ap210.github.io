4 Information requirements

This clause specifies the information requirements for the Design specific assignment to assembly usage view application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Design_specific_assignment_to_assembly_usage_view_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Design_specific_assignment_to_assembly_usage_view_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Assembly_module_with_interconnect_component_arm;    --  ISO/TS 10303-1643

USE FROM Assembly_module_with_subassembly_arm;    --  ISO/TS 10303-1641

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 layered_assembly_module_interface_select   

The layered_assembly_module_interface_select type allows for the designation of the data types Assembly_module_component_terminal, Interconnect_module_component_terminal and Physical_component_interface_terminal.

EXPRESS specification:

*)
TYPE layered_assembly_module_interface_select = SELECT
   (Assembly_module_component_terminal,
    Interconnect_module_component_terminal,
    Physical_component_interface_terminal);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Component_terminal_to_assembly_module_interface_terminal_assignment   

A Component_terminal_to_assembly_module_interface_terminal_assignment is a type of Component_feature_to_physical_usage_view_assignment.

EXPRESS specification:

*)
ENTITY Component_terminal_to_assembly_module_interface_terminal_assignment
  SUBTYPE OF (Component_feature_to_physical_usage_view_assignment);
  SELF\Component_feature_to_physical_usage_view_assignment.defined_part_feature RENAMED usage_view_terminal : Assembly_module_interface_terminal;
  SELF\Component_feature_to_physical_usage_view_assignment.assigned_design_object RENAMED assigned_terminal : layered_assembly_module_interface_select;
UNIQUE
  UR1: usage_view_terminal;
END_ENTITY;
(*

Attribute definitions:

usage_view_terminal: specifies a role of the Assembly_module_interface_terminal for the Component_terminal_to_assembly_module_interface_terminal_assignment.

assigned_terminal: specifies the layered_assembly_module_interface_select for the Component_terminal_to_assembly_module_interface_terminal_assignment.

Formal propositions:

UR1: The usage_view_terminal shall be unique within a population of Component_terminal_to_assembly_module_interface_terminal_assignment.

4.3.2 Component_terminal_to_layered_assembly_module_terminal_assignment   

A Component_terminal_to_layered_assembly_module_terminal_assignment is a type of Component_feature_to_physical_usage_view_assignment. A Component_terminal_to_layered_assembly_module_terminal_assignment is the relationship between a Layered_assembly_module_terminal and one of an Assembly_module_component_terminal, a Physical_component_interface_terminal, or an Interconnect_module_component_terminal, to provide traceability from the data in the design view to the data in the usage view for the design organization for archival purposes.

EXPRESS specification:

*)
ENTITY Component_terminal_to_layered_assembly_module_terminal_assignment
  SUBTYPE OF (Component_feature_to_physical_usage_view_assignment);
  SELF\Component_feature_to_physical_usage_view_assignment.defined_part_feature RENAMED usage_view_terminal : Layered_assembly_module_terminal;
  SELF\Component_feature_to_physical_usage_view_assignment.assigned_design_object RENAMED assigned_terminal : layered_assembly_module_interface_select;
UNIQUE
  UR1: usage_view_terminal;
END_ENTITY;
(*

Attribute definitions:

usage_view_terminal: specifies a role of the Layered_assembly_module_terminal for the Component_terminal_to_layered_assembly_module_terminal_assignment.

assigned_terminal: specifies one of a layered_assembly_module_interface_select for the Component_terminal_to_layered_assembly_module_terminal_assignment.

Formal propositions:

UR1: The usage_view_terminal shall be unique within a population of Component_terminal_to_layered_assembly_module_terminal_assignments.

4.4 ARM subtype constraint definitions

This subclause specifies the ARM subtype constraints for this module. Each subtype constraint places constraints on the possible super-type / subtype instantiations. The ARM subtype constraints and definitions are specified below.

4.4.1 dsatauv_component_feature_to_physical_usage_view_assignment_subtypes   

The dsatauv_component_feature_to_physical_usage_view_assignment_subtypes constraint specifies a constraint that applies to instances of subtypes of Component_feature_to_physical_usage_view_assignment.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT dsatauv_component_feature_to_physical_usage_view_assignment_subtypes FOR Component_feature_to_physical_usage_view_assignment;
  ONEOF (Component_terminal_to_assembly_module_interface_terminal_assignment,
         Component_terminal_to_layered_assembly_module_terminal_assignment);
END_SUBTYPE_CONSTRAINT;
(*

4.4.2 dsatauv_physical_component_terminal_subtypes   

The dsatauv_physical_component_terminal_subtypes constraint specifies a constraint that applies to instances of subtypes of Physical_component_terminal.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT dsatauv_physical_component_terminal_subtypes FOR Physical_component_terminal;
  ONEOF (Assembly_module_component_terminal,
         Bare_die_component_terminal,
         Interconnect_component_join_terminal,
         Interconnect_module_component_terminal,
         Minimally_defined_component_terminal,
         Packaged_component_join_terminal);
END_SUBTYPE_CONSTRAINT;
