4 Information requirements

This clause specifies the information requirements for the Test requirement allocation application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Test_requirement_allocation_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Test_requirement_allocation_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Functional_assignment_to_part_arm;    --  ISO/TS 10303-1674

USE FROM Requirement_decomposition_arm;    --  ISO/TS 10303-1740

USE FROM Functional_usage_view_arm;    --  ISO/TS 10303-1705
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Functional_assignment_to_part_arm ISO/TS 10303-1674
Requirement_decomposition_arm ISO/TS 10303-1740
Functional_usage_view_arm ISO/TS 10303-1705

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 tra_requirement_assignment_item   

The tra_requirement_assignment_item type is an extension of the requirement_assignment_item type. It adds the data type Product_version to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE tra_requirement_assignment_item = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON requirement_assignment_item WITH
   (Product_version);
END_TYPE;
(*

4.3 ARM entity definition

This subclause specifies the ARM entity for this module. The ARM entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entity and definition is specified below.

4.3.1 Test_requirement_allocation   

A Test_requirement_allocation is a type of Requirement_assignment for which the Predefined_requirement_view_definition has test characteristics. The terminals of the product are related to either parametric data in an executable form or to data derived from experimental or theoretical measurements.

NOTE    The structural split of allocation and requirement occurrence permits re-use of common test requirements and associated procedures over multiple products.

EXPRESS specification:

*)
ENTITY Test_requirement_allocation
  SUBTYPE OF (Requirement_assignment);
  SELF\Requirement_assignment.assigned_to : Product_version;
  analytical_model_based_test_access : OPTIONAL SET[1:?] OF Analytical_model_port_assignment;
  functional_specification_based_test_access : OPTIONAL SET[1:?] OF Functional_usage_view_to_part_terminal_assignment;
  SELF\Requirement_assignment.assigned_requirement RENAMED assigned_requirement_property : Predefined_requirement_view_definition;
WHERE
  WR1: EXISTS(analytical_model_based_test_access) XOR EXISTS(functional_specification_based_test_access);
  WR2: NOT (EXISTS(functional_specification_based_test_access)) OR (SIZEOF(QUERY(fa <* functional_specification_based_test_access | NOT (fa.physical_usage_view_terminal\Shape_element.containing_shape.defined_version = SELF\Requirement_assignment.assigned_to))) = 0);
  WR3: assigned_requirement_property\Product_view_definition.initial_context\View_definition_context.life_cycle_stage = 'test';
END_ENTITY;
(*

Attribute definitions:

assigned_to: an attribute inherited from the Requirement_assignment shall be redeclared as the Product_version for the Test_requirement_allocation. The assigned_to is a specialization of the attribute inherited from Requirement_assignment.

analytical_model_based_test_access: specifies a role of the Analytical_model_port_assignment for the Test_requirement_allocation. The analytical_model_based_test_access specifies the mapping of the ports of the Analytical_model_definition onto the relevant terminals of the product for which the test is specified. There shall exist more than one Analytical_model_port_assignment for the Test_requirement_allocation. The value of this attribute need not be specified.

NOTE    The mapping provides a formal mechanism for associating the details of the required behavior represented in computer executable form, as defined in the body of an Analytical_model_definition, with the terminals of the tested product. The explicit assignment of values to the parameters of the Analytical_model_definition is supported.

functional_specification_based_test_access: specifies a role of the Functional_usage_view_to_part_terminal_assignment for the Test_requirement_allocation. The functional_specification_based_test_access specifies the mapping of the terminals of the Functional_unit_usage_view onto the relevant terminals of the product for which the test is specified. There shall exist more than one Functional_usage_view_to_part_terminal_assignment for the Test_requirement_allocation. The value of this attribute need not be specified.

NOTE    The mapping provides a formal mechanism for associating the details of the required behavior represented as a result of an experiment, as defined by a Functional_specification, with the terminals of the tested product.

assigned_requirement_property: specifies a role of Predefined_requirement_view_definition for the Test_requirement_allocation.

Formal propositions:

WR1: IF analytical_model_based_test_access is provided THEN the version for the associated definition of the source of the analytical_model_based_test_access shall be the version assigned_to inherited from the Requirement_assignment.

WR2: IF analytical_model_based_test_access is provided THEN all members of analytical_model_based_test_access shall reference the same Analytical_model_definition that is referenced by the assigned_requirement attribute inherited from the Requirement_assignment.

WR3: Each Test_requirement_allocation shall reference a Predefined_requirement_view_definition with a life cycle context of test.



*)
END_SCHEMA;  -- Test_requirement_allocation_arm
