4 Information requirements

This clause specifies the information requirements for the Assembly 2D shape application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Assembly_2d_shape_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Assembly_2d_shape_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Assembly_technology_arm;    --  ISO/TS 10303-1649

USE FROM Physical_unit_2d_design_view_arm;    --  ISO/TS 10303-1724

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Assembly_technology_arm ISO/TS 10303-1649
Physical_unit_2d_design_view_arm ISO/TS 10303-1724

4.2 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.2.1 Bond_assembly_2d_position   

A Bond_assembly_2d_position is a means to provide information about the shape and position of a bond in an assembly in two-dimensional space.

EXPRESS specification:

*)
ENTITY Bond_assembly_2d_position;
  bond : Assembled_with_bonding;
  assembly_model : Physical_unit_planar_shape_model;
  bond_model : Default_assembly_bond_shape_model;
  bond_definition_placement : Axis_placement_2d;
UNIQUE
  UR1: bond;
END_ENTITY;
(*

Attribute definitions:

bond: specifies the role of the Assembled_with_bonding for the Bond_assembly_2d_position. The bond is the joint that is created in the assembly.

assembly_model: specifies the role of the Physical_unit_planar_shape_model for the Bond_assembly_2d_position. The assembly_model is the planar shape of the bond.

bond_model: specifies the role of the Default_assembly_bond_shape_model for the Bond_assembly_2d_position. The bond_model is the default shape of the bond.

bond_definition_placement: specifies the role of the Axis_placement_2d for the Bond_assembly_2d_position. The bond_definition_placement is the position of the bond in two-dimensional space.

Formal propositions:

UR1: The bond shall be unique within a population of Bond_assembly_2d_position.

4.2.2 Component_2d_edge_location   

A Component_2d_edge_location is a type of Component_2d_location. The Component_2d_edge_location places components on the edge of a substrate in order to maintain contact with both the top and bottom circuitry.

NOTE    Connection_zone_bare_die_interface_plane_relationship and Connection_zone_based_assembly_joint may be used to provide more details if needed by a receiving organization.

EXPRESS specification:

*)
ENTITY Component_2d_edge_location
  SUBTYPE OF (Component_2d_location);
  reference_terminal_assembly_joint : Assembly_joint;
  mounting_surface_assembly_joint : Assembly_joint;
WHERE
  WR1: reference_terminal_assembly_joint <> mounting_surface_assembly_joint;
  WR2: SELF\Component_2d_location.substrate_location = FALSE;
  WR3: SIZEOF(['ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_TERMINAL', 'ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_SURFACE_FEATURE'] * TYPEOF(mounting_surface_assembly_joint.assembly_feature_1)) >= 1;
  WR4: 'PACKAGE_ARM.PRIMARY_REFERENCE_TERMINAL' IN TYPEOF(reference_terminal_assembly_joint\assembly_joint.assembly_feature_1);
  WR5: NOT EXISTS(SELF\Representation_relationship.description);
END_ENTITY;
(*

Attribute definitions:

reference_terminal_assembly_joint: specifies one role of the Assembly_joint for the Component_2d_edge_location. The reference_terminal_assembly_joint joins the Primary_reference_terminal of the component being located to the associated mating component feature. The surface that the mating component feature is associated with is part of the orientation description for the Component_2d_edge_location.

mounting_surface_assembly_joint: specifies one role of the Assembly_joint for the Component_2d_edge_location.

Formal propositions:

WR1: reference_terminal_assembly_joint shall not be mounting_surface_assembly_joint.

WR2: The substrate_location inherited from the Component_2d_location shall be "False".

WR3: assembly_feature_1 of mounting_surface_assembly_joint shall only be a Interconnect_module_component_terminal or a Interconnect_module_component_surface_feature.

WR4: The reference_terminal_assembly_joint shall reference an entity instance of type Primary_reference_terminal as assembly_feature_1.

WR5: The description shall not be populated.

4.2.3 Component_2d_stacked_location   

A Component_2d_stacked_location is a type of Component_2d_location where the components are vertically stacked. The stack reference direction shall be parallel to the normal vector of the primary surface of the interconnect.

NOTE 1   The reference direction is independent of whether the stack is based on the primary or secondary surface of the interconnect.

NOTE 2   The reference direction is opposite to that of the Z axis of the interconnect internal layer stackup.

NOTE 3   Applications should ensure that stacked components maintain a consistent direction in the stack.

EXAMPLE    The arrangement of stacked memory chips may be represented using Component_2d_stacked_location.

EXPRESS specification:

*)
ENTITY Component_2d_stacked_location
  SUBTYPE OF (Component_2d_location);
  mounting_joint : Assembly_joint;
WHERE
  WR1: SIZEOF(['ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_TERMINAL', 'ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_SURFACE_FEATURE'] * TYPEOF(mounting_joint.assembly_feature_1)) = 0;
  WR2: SIZEOF(['ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_TERMINAL', 'ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_SURFACE_FEATURE'] * TYPEOF(mounting_joint.assembly_feature_2)) = 0;
  WR3: SELF\Component_2d_location.substrate_location = FALSE;
  WR4: NOT EXISTS(SELF\Representation_relationship.description);
END_ENTITY;
(*

Attribute definitions:

mounting_joint: specifies the role of the Assembly_joint for the Component_2d_stacked_location. The joint provides a mechanism for traversing the vertical stack unambiguously. The stack direction vector origin shall be represented by the feature specified by the assembly_feature_1 attribute of the mounting_joint. The stack direction vector destination shall be represented by the feature specified by the assembly_feature_2 attribute of the mounting_joint.

Formal propositions:

WR1: assembly_feature_1 of mounting_joint shall not be a Interconnect_module_component_terminal nor a Interconnect_module_component_surface_feature.

WR2: assembly_feature_2 of mounting_joint shall not be a Interconnect_module_component_terminal nor a Interconnect_module_component_surface_feature.

WR3: The substrate_location inherited from the Component_2d_location shall be "False".

WR4: The description shall not be populated.

4.2.4 Component_2d_surface_location   

A Component_2d_surface_location is a type of Component_2d_location. The Component_2d_surface_location places components on the external surface of a substrate.

NOTE    Connection_zone_bare_die_interface_plane_relationship and Connection_zone_based_assembly_joint may be used to provide more details if needed by a receiving organization.

EXPRESS specification:

*)
ENTITY Component_2d_surface_location
  SUBTYPE OF (Component_2d_location);
  mounting_surface_assembly_joint : Assembly_joint;
WHERE
  WR1: SELF\Component_2d_location.substrate_location = FALSE;
  WR2: SIZEOF(['ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_TERMINAL', 'ASSEMBLY_MODULE_WITH_INTERCONNECT_COMPONENT_ARM.INTERCONNECT_MODULE_COMPONENT_SURFACE_FEATURE'] * TYPEOF(mounting_surface_assembly_joint.assembly_feature_1)) >= 1;
  WR3: NOT EXISTS(SELF\Representation_relationship.description);
END_ENTITY;
(*

Attribute definitions:

mounting_surface_assembly_joint: specifies the Assembly_joint that joins the Component_mounting_feature of the component being located to the associated mating component feature. The surface the mating component feature is associated with is part of the orientation description for the Component_2d_surface_location.

Formal propositions:

WR1: The substrate_location inherited from the Component_2d_location shall be "False".

WR2: assembly_feature_1 of mounting_surface_assembly_joint shall only be a Interconnect_module_component_terminal or a Interconnect_module_component_surface_feature.

WR3: The description shall not be populated.

4.3 ARM subtype constraint definition

This subclause specifies the ARM subtype constraint for this module. The subtype constraint places a constraint on the possible super-type / subtype instantiations. The ARM subtype constraint and definition is specified below.

4.3.1 a2ds_component_2d_location_subtypes   

The a2ds_component_2d_location_subtypes constraint specifies a constraint that applies to instances of subtypes of Component_2d_location.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT a2ds_component_2d_location_subtypes FOR Component_2d_location;
  ONEOF (Component_2d_edge_location,
         Component_2d_stacked_location,
         Component_2d_surface_location);
END_SUBTYPE_CONSTRAINT;
