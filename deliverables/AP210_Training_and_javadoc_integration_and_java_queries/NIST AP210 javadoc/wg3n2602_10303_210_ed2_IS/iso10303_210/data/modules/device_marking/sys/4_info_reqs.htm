4 Information requirements

This clause specifies the information requirements for the Device marking application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Device_marking_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Device_marking_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Approval_arm;    --  ISO/TS 10303-1012

USE FROM Layered_interconnect_complex_template_arm;    --  ISO/TS 10303-1716

USE FROM Part_template_non_planar_shape_arm;    --  ISO/TS 10303-1719

REFERENCE FROM Specification_document_arm    --  ISO/TS 10303-1747
  (get_document_definition);
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Approval_arm ISO/TS 10303-1012
Layered_interconnect_complex_template_arm ISO/TS 10303-1716
Part_template_non_planar_shape_arm ISO/TS 10303-1719
Specification_document_arm ISO/TS 10303-1747

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.2.1 dm_documented_element_select   

The dm_documented_element_select type is an extension of the documented_element_select type. It adds the data type Marking to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE dm_documented_element_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON documented_element_select WITH
   (Marking);
END_TYPE;
(*

4.2.2 dm_material_item_select   

The dm_material_item_select type is an extension of the material_item_select type. It adds the data type Marking to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE dm_material_item_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON material_item_select WITH
   (Marking);
END_TYPE;
(*

4.2.3 marked_message_select   

The marked_message_select type allows for the designation of the data types Configuration_controlled_message and Text_representation_item.

EXPRESS specification:

*)
TYPE marked_message_select = SELECT
   (Configuration_controlled_message,
    Text_representation_item);
END_TYPE;
(*

4.2.4 marking_shape_select   

The marking_shape_select type allows for the designation of the data types Part_template_3d_shape_model, Part_template_non_planar_2d_shape_model, Part_template_planar_shape_model, Physical_unit_3d_shape_model and Physical_unit_planar_shape_model.

EXPRESS specification:

*)
TYPE marking_shape_select = SELECT
   (Part_template_3d_shape_model,
    Part_template_non_planar_2d_shape_model,
    Part_template_planar_shape_model,
    Physical_unit_3d_shape_model,
    Physical_unit_planar_shape_model);
END_TYPE;
(*

4.2.5 message_source_select   

The message_source_select type allows for the designation of the data types Approval, Organization and Product_version.

EXPRESS specification:

*)
TYPE message_source_select = SELECT
   (Approval,
    Organization,
    Product_version);
END_TYPE;
(*

4.2.6 message_type   

A message_type lists the types of the message.

EXPRESS specification:

*)
TYPE message_type = ENUMERATION OF
   (part_number,
    manufacturer_code,
    revision_identifier,
    approval_date);
END_TYPE;
(*

Enumerated item definitions:

part_number: specifies a part number;

manufacturer_code: specifies a manufacturer code;

revision_identifier: specifies a revision identifier;

approval_date: specifies an approval date.

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Configuration_controlled_message   

A Configuration_controlled_message is a message that has its content under version control, and that is source data for application of marking materials to a product.

EXPRESS specification:

*)
ENTITY Configuration_controlled_message;
  message_source : message_source_select;
  message_content : STRING;
  message_type_specification : message_type;
END_ENTITY;
(*

Attribute definitions:

message_source: specifies one of the message_source_select for the Configuration_controlled_message.

message_content: specifies the text in the Configuration_controlled_message.

message_type_specification: specifies one of the message_types for the Configuration_controlled_message. A message_type_specification is necessary to differentiate among the possible message sources to identify the one that has been selected for the literal string.

4.3.2 Marking   

A Marking is the specification of what, where, and how information that is placed on a Part_view_definition for identification purposes is applied. Marking in this part of ISO 10303 is considered to not be part of the definition of a Stratum. Marking is text that is applied to a material object. The message_representation_text shall be an item in the message_representation.

EXPRESS specification:

*)
ENTITY Marking;
  marked_message : marked_message_select;
  marked_device_shape : marking_shape_select;
  placement : Axis_placement;
  message_representation : Geometric_model;
  message_representation_text : Text_literal_with_extent;
DERIVE
  marking_specification : SET[0:?] OF Document_definition := get_document_definition(SELF, 'marking specification', 'SPECIFICATION_DOCUMENT_ARM.SPECIFICATION_DEFINITION');
INVERSE
  marking_material : Material_identification FOR items;
WHERE
  WR1: SIZEOF(QUERY(st <* message_representation\Geometric_model.items | (st = message_representation_text))) = 1;
  WR2: SIZEOF(QUERY(ap <* marked_device_shape\Geometric_model.items | (ap = placement))) = 1;
  WR3: SIZEOF(TYPEOF(marked_device_shape) * ['DEVICE_MARKING_ARM.PHYSICAL_UNIT_3D_SHAPE_MODEL', 'DEVICE_MARKING_ARM.PHYSICAL_UNIT_PLANAR_SHAPE_MODEL']) = 1;
  WR4: 'CONDUCTIVITY_MATERIAL_ASPECTS_ARM.'+ 'MATERIAL_IDENTIFICATION_WITH_CONDUCTIVITY_CLASSIFICATION' IN TYPEOF (marking_material);
  WR5: EXISTS (marking_specification) AND (SIZEOF(marking_specification) = 1);
END_ENTITY;
(*

Attribute definitions:

marked_message: specifies the marked_message_select for the Marking. The marked_message specifies either a Configuration_controlled_message or Text_representation_item for Marking.

marked_device_shape: specifies the marking_shape_select for the Marking. The marked_device_shape specifies one of a Part_template_3d_shape_model, a Physical_unit_3d_shape_model, a Part_template_non_planar_2d_shape_model, a Part_template_planar_shape_model or a Physical_unit_planar_shape_model that is the shape definition for the physical device that receives the Marking.

placement: specifies the Axis_placement of the Marking. The placement specifies the placement of the message_representation in the marked_device_shape.

message_representation: specifies role of the Geometric_model for the Marking.

message_representation_text: specifies role of the Text_literal_with_extent for the Marking.

marking_specification: specifies the Document_definition of the process and format for properly applying the Marking.

marking_material: specifies an inverse relationship that specifies that the existence of the Marking is dependent on the existence of the Material_identification that specifies the Marking as its items.

Formal propositions:

WR1: Exactly one member of items of message_representation shall be the message_representation_text.

WR2: Exactly one member of items of marked_device_shape shall be the placement.

WR3: The marked_device_shape shall be either a Physical_unit_3d_shape_model or shall be a Physical_unit_planar_shape_model.

WR4: The marking_material shall be of type Material_identification_with_conductivity_classification.

WR5: The marking_specification shall be provided and its size shall be one.



*)
END_SCHEMA;  -- Device_marking_arm
