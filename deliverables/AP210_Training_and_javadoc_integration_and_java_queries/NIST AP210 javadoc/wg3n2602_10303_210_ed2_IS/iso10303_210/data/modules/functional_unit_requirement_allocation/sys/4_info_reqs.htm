4 Information requirements

This clause specifies the information requirements for the Functional unit requirement allocation application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Functional_unit_requirement_allocation_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Functional_unit_requirement_allocation_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Network_functional_design_view_arm;    --  ISO/TS 10303-1704
(*

Network_functional_design_view_arm ISO/TS 10303-1704

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 fura_requirement_assignment_item   

The fura_requirement_assignment_item type is an extension of the requirement_assignment_item type. It adds the data type Functional_unit to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE fura_requirement_assignment_item = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON requirement_assignment_item WITH
   (Functional_unit);
END_TYPE;
(*

4.3 ARM entity definition

This subclause specifies the ARM entity for this module. The ARM entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entity and definition is specified below.

4.3.1 Functional_unit_requirement_allocation   

A Functional_unit_requirement_allocation is a type of Requirement_assignment. A Functional_unit_requirement_allocation is the relationship between the Predefined_requirement_view_definition and the Functional_unit that satisfies the Predefined_requirement_view_definition.

EXPRESS specification:

*)
ENTITY Functional_unit_requirement_allocation
  SUBTYPE OF (Requirement_assignment);
  SELF\Requirement_assignment.assigned_to : Functional_unit;
WHERE
  WR1: NOT EXISTS (SELF\Requirement_assignment.description);
END_ENTITY;
(*

Attribute definitions:

assigned_to: an attribute inherited from the Requirement_assignment supertype specifies the Functional_unit that satisfies the Predefined_requirement_view_definition for the Functional_unit_requirement_allocation.

Formal propositions:

WR1: The description shall not be populated.



*)
END_SCHEMA;  -- Functional_unit_requirement_allocation_arm
