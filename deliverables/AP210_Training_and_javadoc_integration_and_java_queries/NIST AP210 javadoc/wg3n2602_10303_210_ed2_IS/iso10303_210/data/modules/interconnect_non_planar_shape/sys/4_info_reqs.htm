4 Information requirements

This clause specifies the information requirements for the Interconnect non planar shape application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Interconnect_non_planar_shape_arm schema and identifies the necessary external references.

*)

4.2 ARM entity definition

This subclause specifies the ARM entity for this module. The ARM entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entity and definition is specified below.

4.2.1 Component_part_2d_non_planar_geometric_representation_relationship   

A Component_part_2d_non_planar_geometric_representation_relationship is a type of Geometric_model_relationship. A Component_part_2d_non_planar_geometric_representation_relationship is the association between a Part_template_planar_shape_model or a Physical_unit_planar_shape_model and the Assembly_component_non_planar_shape_model that is the occurrence of that planar shape. If the type of the shape representation of the assembly is different from the type of the shape representation of the part, the Component_part_2d_non_planar_geometric_representation_relationship allows a processor to create a shape rep of the assembly type and establish an equivalency relationship between that shape representation and the part shape representation. If the shape representation of the assembly is of the same type as the part shape representation, the Component_part_2d_non_planar_geometric_representation_relationship shall reference the same shape representation as both the part_shape and the component_shape. That instance shall be a complex instance of the part shape and of the component shape.

EXPRESS specification:

*)
ENTITY Component_part_2d_non_planar_geometric_representation_relationship
  SUBTYPE OF (Geometric_model_relationship);
  SELF\Geometric_model_relationship.rep_1 RENAMED part_shape : Part_template_planar_shape_model;
  SELF\Geometric_model_relationship.rep_2 RENAMED component_shape : Assembly_component_non_planar_shape_model;
WHERE
  WR1: NOT EXISTS(SELF\Representation_relationship.relation_type);
END_ENTITY;
(*

Attribute definitions:

part_shape: specifies the role of the Part_template_planar_shape_model for the Component_part_2d_non_planar_geometric_representation_relationship.

component_shape: specifies the role of the Assembly_component_non_planar_shape_model for the Component_part_2d_non_planar_geometric_representation_relationship.

Formal propositions:

WR1: The relation_type shall not be populated.



*)
END_SCHEMA;  -- Interconnect_non_planar_shape_arm
