5.2 MIM EXPRESS short listing

This clause specifies the EXPRESS schema derived from the mapping table. It uses elements from the common resources or from other application modules and defines the EXPRESS constructs that are specific to this part of ISO 10303.

This clause constitutes the Module Interpreted Module (MIM) of the application module.

This clause also specifies the modifications that apply to the constructs imported from the common resources.

The following restrictions apply to the use, in this schema, of constructs defined in common resources or in application modules:

  • use of a supertype entity does not make applicable any of its specializations, unless the specialization is also imported in the MIM schema;
  • use of a SELECT type does not make applicable any of its listed types unless the listed type is also imported in the MIM schema.

NOTE 1   This schema is upward compatible, for their common scope areas, with ISO 10303-214:2003, with the following exceptions:

  • the concept of product class has a single mapping in the present application module, onto the entity product_class;
  • the concept of product identification is only mapped in the present application module, onto the entities product_identification and product_specification;
  • the concept of general classification defined in the ARM of ISO 10303-214:2003 has a single mapping in the present application module, onto the entity characterized_class;
  • the concept of physical model is mapped onto a subtype of product_definition and not of product_definition_with_associated_documents.

NOTE 2   This clause specifies rules that apply to the entity data types defined or imported through USE FROM statements. In some cases, these rules do not correspond to constraints specified in the ARM of the present module but reflect constraints specified in imported modules. For example, most of the dependent instantiable rules are related to the fact that in the modules where the constrained MIM entity is defined, the corresponding construct in the ARM is an attribute and not an entity. That is, in particular, the case for many role attributes. However, as the constrained MIM entity may be used in various modules and with various constraints, it is only at the level of the application protocol or of a conformance class that a formal rule applied to the MIM entity can be specified, synthesising the constraints applied to this MIM entity in all the imported modules.
