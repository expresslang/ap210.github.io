4 Information requirements

This clause specifies the information requirements for the Interconnect module to assembly module relationship application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Interconnect_module_to_assembly_module_relationship_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Interconnect_module_to_assembly_module_relationship_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Interconnect_module_connection_routing_arm;    --  ISO/TS 10303-1684
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Interconnect_module_connection_routing_arm ISO/TS 10303-1684

NOTE 2   See Annex C, Figures C.1, C.2and C.3 for a graphical representation of this schema.

4.2 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.2.1 component_based_message_type   

A component_based_message_type lists the types of the component based message.

EXPRESS specification:

*)
TYPE component_based_message_type = ENUMERATION OF
   (component_reference_designation,
    terminal_designation);
END_TYPE;
(*

Enumerated item definitions:

component_reference_designation: specifies reference designator of the component;

terminal_designation: specifies reference designator of the terminal.

4.2.2 component_or_component_feature   

The component_or_component_feature type allows for the designation of the data types Assembly_component, Component_external_reference, Component_feature and Component_feature_external_reference.

EXPRESS specification:

*)
TYPE component_or_component_feature = SELECT
   (Assembly_component,
    Component_external_reference,
    Component_feature,
    Component_feature_external_reference);
END_TYPE;
(*

4.2.3 imtar_bare_die_component_or_packaged_component   

The imtar_bare_die_component_or_packaged_component type is an extension of the bare_die_component_or_packaged_component type. It adds the data types Bare_die_component and Packaged_component to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

*)

    Packaged_component);
END_TYPE;
(*

4.2.4 imtar_bare_die_terminal_or_packaged_part_terminal   

The imtar_bare_die_terminal_or_packaged_part_terminal type is an extension of the bare_die_terminal_or_packaged_part_terminal type. It adds the data types Bare_die_terminal and Packaged_part_terminal to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE imtar_bare_die_terminal_or_packaged_part_terminal = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON bare_die_terminal_or_packaged_part_terminal WITH

    Packaged_part_terminal);
END_TYPE;
(*

4.2.5 imtar_documented_element_select   

The imtar_documented_element_select type is an extension of the documented_element_select type. It adds the data type component_or_component_feature to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Assembly_requirement_for_interconnect_relationship   

An Assembly_requirement_for_interconnect_relationship is a type of Part_definition_relationship.

EXPRESS specification:

*)
ENTITY Assembly_requirement_for_interconnect_relationship
  SUBTYPE OF (Part_definition_relationship);
  SELF\View_definition_relationship.relating_view : Assembly_module_design_view;
  SELF\View_definition_relationship.related_view : Layered_interconnect_module_design_view;
END_ENTITY;
(*

Attribute definitions:

relating_view: one of the instances of Assembly_module_design_view that is part of the relationship.

related_view: the other instance of Layered_interconnect_module_design_view that is part of the relationship. If one element of the relationship is dependent upon the other, this attribute shall be the dependent one.

4.3.2 Component_based_annotation_symbol_placement_relationship   

A Component_based_annotation_symbol_placement_relationship is a type of Component_feature_to_layout_feature_relationship. A Component_based_annotation_symbol_placement_relationship is the information that identifies a special_symbol_component and is to be placed in the context of the design. This information overrides any such information that may be provided in the library.

EXPRESS specification:

*)
ENTITY Component_based_annotation_symbol_placement_relationship
  SUBTYPE OF (Component_feature_to_layout_feature_relationship);
  SELF\Component_feature_to_layout_feature_relationship.related_stratum_concept : special_symbol_component;
END_ENTITY;
(*

Attribute definitions:

related_stratum_concept: the second instance of special_symbol_component that is part of the relationship.

4.3.3 Component_based_annotation_text_relationship   

A Component_based_annotation_text_relationship is a type of Component_feature_to_layout_feature_relationship. A Component_based_annotation_text_relationship is the information that specifies an explicit text message based on the component, where the message content is either the value of the reference to the component's locator or the value of the terminal pin identifier.

NOTE    This information permits product data stored in standard exchange files to serve as messages that are presented to the user in the medium of the Stratum, such as the text of the locator reference or the terminal pin name. This information is not intended to be populated for general etched characters on the board.

EXPRESS specification:

*)
ENTITY Component_based_annotation_text_relationship
  SUBTYPE OF (Component_feature_to_layout_feature_relationship);
  SELF\Component_feature_to_layout_feature_relationship.related_stratum_concept : Laminate_text_string_component;
  message_type : component_based_message_type;
END_ENTITY;
(*

Attribute definitions:

related_stratum_concept: the second instance of Laminate_text_string_component that is part of the relationship. It redeclares the related_stratum_concept inherited from Component_feature_to_layout_feature_relationship to be Laminate_text_string_component for the Component_based_annotation_text_relationship. There is no explicit geometric relationship defined in this part of ISO 10303 between the design element referenced by the message presented by the related_stratum_concept and the related_stratum_concept itself. Only the related_stratum_concept is positioned in the current design. The component or feature referenced by the message in the attribute related_stratum_concept is defined in a different design that is an Assembly_module_design_view.

NOTE    This is the actual physical message, as realized by the fabrication process. The position of the corresponding elements in the two designs are usually defined by an end user using a layout tool which presents a view of the data from the assembly in the layout tool for design efficiency.

message_type: specifies either a component locator reference or a terminal locator for the Component_based_annotation_text_relationship. The message_type is a discriminator for determining which data to use from the attribute relating_feature that is inherited from the Application Object Component_feature_to_layout_feature_relationship. The useful options are either External_item_identification or External_item_identification. For the component locator reference option, two additional queries are required, one to determine the Next_assembly_usage and the second to determine the Assembly_component. The component locator reference is the attribute location_indicator of the returned instance of Next_assembly_usage. The string content of that attribute shall be the string content of the message of the related_stratum_concept. For the terminal locator option, additional queries are necessary, continuing with the Assembly_component and terminating at the relevant Part_terminal_external_reference with an intermediate query of Component_feature. The string content of the name attribute of Part_terminal_external_reference shall be the string content of the message of the related_stratum_concept.

NOTE 1   The attribute message_type is not a value of data. It is a declaration of the type of Component_based_annotation_text_relationship.

NOTE 2   The data contained in the location_indicator attribute in a member of Next_assembly_usage is an instance of a message of one type.

NOTE 3   The data contained in the name attribute in a member of Package_terminal is an instance of a message of the second type.

4.3.4 Component_feature_to_layout_feature_relationship   

A Component_feature_to_layout_feature_relationship is the association between a component or feature of a component and a layout feature, based upon a requirement. All features related with this relationship shall be in the same design unless the reference is to a External_item_identification or to a External_item_identification. A Component_feature_to_layout_feature_relationship may be either a Component_based_annotation_symbol_placement_relationship or a Component_based_annotation_text_relationship.

EXPRESS specification:

*)
ENTITY Component_feature_to_layout_feature_relationship
  SUPERTYPE OF (ONEOF (Component_based_annotation_symbol_placement_relationship,
                       Component_based_annotation_text_relationship));
  relating_feature : component_or_component_feature;
  related_stratum_concept : stratum_concept;
INVERSE
  requirement : SET[0:1] OF Requirement_assignment FOR assigned_to;
END_ENTITY;
(*

Attribute definitions:

relating_feature: an instance that specifies the component_or_component_feature for the Component_feature_to_layout_feature_relationship.

related_stratum_concept: an instance that specifies the stratum_concept to which the relating_feature is related for the Component_feature_to_layout_feature_relationship.

requirement: an inverse relationship that specifies that the existence of the Component_feature_to_layout_feature_relationship is dependent on the existence of the Requirement_assignment that specifies the Component_feature_to_layout_feature_relationship as its assigned_to.



*)
END_SCHEMA;  -- Interconnect_module_to_assembly_module_relationship_arm
