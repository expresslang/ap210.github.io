4 Information requirements

This clause specifies the information requirements for the Pre defined datum 3D symbol application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Pre_defined_datum_3d_symbol_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Pre_defined_datum_3d_symbol_arm;

4.2 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.2.1 Pre_defined_datum_axis_3d_symbol   

A Pre_defined_datum_axis_3d_symbol is a type of Geometric_model. A Pre_defined_datum_axis_3d_symbol is a standard representation for a Datum_axis in a 3 dimensional coordinate system.

EXPRESS specification:

*)
ENTITY Pre_defined_datum_axis_3d_symbol
  SUBTYPE OF (Geometric_model);
  associated_element : Datum_axis;
WHERE
  WR1: SELF\Geometric_model.context_of_items.dimension_count = 3;
  WR2: NOT EXISTS(SELF\Representation.name);
  WR3: NOT EXISTS(SELF\Representation.description);
END_ENTITY;
(*

Attribute definitions:

associated_element: specifies a role of the Datum_axis for the Pre_defined_datum_axis_3d_symbol.

Formal propositions:

WR1: The context of Pre_defined_datum_axis_3d_symbol shall be three-dimensional.

WR2: The name shall not be populated.

WR3: The description shall not be populated.

4.2.2 Pre_defined_datum_plane_3d_symbol   

A Pre_defined_datum_plane_3d_symbol is a type of Geometric_model. A Pre_defined_datum_plane_3d_symbol is a standard representation for a Datum_plane in a 3 dimensional coordinate system. A Pre_defined_datum_plane_3d_symbol shall be parallel with the normal vector of the plane, and shall intersect the plane at the origin of the symbol.

EXPRESS specification:

*)
ENTITY Pre_defined_datum_plane_3d_symbol
  SUBTYPE OF (Geometric_model);
  associated_element : Datum_plane;
WHERE
  WR1: SELF\Geometric_model.context_of_items.dimension_count = 3;
  WR2: NOT EXISTS(SELF\Representation.name);
  WR3: NOT EXISTS(SELF\Representation.description);
END_ENTITY;
(*

Attribute definitions:

associated_element: specifies a role of the Datum_plane for the Pre_defined_datum_plane_3d_symbol.

Formal propositions:

WR1: The context of Pre_defined_datum_plane_3d_symbol shall be three-dimensional.

WR2: The name shall not be populated.

WR3: The description shall not be populated.

4.3 ARM subtype constraint definition

This subclause specifies the ARM subtype constraint for this module. The subtype constraint places a constraint on the possible super-type / subtype instantiations. The ARM subtype constraint and definition is specified below.

4.3.1 pdd3ds_geometric_model_subtypes   

The pdd3ds_geometric_model_subtypes constraint specifies a constraint that applies to instances of subtypes of Geometric_model.

EXPRESS specification:

*)
SUBTYPE_CONSTRAINT pdd3ds_geometric_model_subtypes FOR Geometric_model;
  ONEOF (Pre_defined_datum_axis_3d_symbol,
         Pre_defined_datum_plane_3d_symbol);
END_SUBTYPE_CONSTRAINT;
