4 Information requirements

This clause specifies the information requirements for the Procedural solid model application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Procedural_solid_model_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Procedural_solid_model_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Procedural_shape_model_arm;    --  ISO/TS 10303-1317
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Procedural_shape_model_arm ISO/TS 10303-1317

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definition

This subclause specifies the ARM type for this application module. The ARM type and definition is specified below.

4.2.1 solid_select   

A solid_select is a select type used to specify the base_solid as the starting point of a shape modification operation.

EXPRESS specification:

*)
TYPE solid_select = SELECT
   (Detailed_geometric_model_element,
    Modified_solid);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Modified_solid   

A Modified_solid is a type of Detailed_geometric_model_element that specifies an shape modification operation to create a local modification of a base_solid.

NOTE    This entity is incompletely modelled in the ARM. Complete specification can be obtained in the definition of the corresponding MIM entity.

EXPRESS specification:

*)
ENTITY Modified_solid
  SUBTYPE OF (Detailed_geometric_model_element);
  rationale : STRING;
  base_solid : solid_select;
END_ENTITY;
(*

Attribute definitions:

rationale: the description of the reasons for using an instance of this entity.

base_solid: the initial solid before the shape modification operation specified by the instantiated instance of a subtype of this entity.

NOTE    The particular modification operation is defined in detail by the subtypes of Modified_solid, the supertype is semantically incomplete.

4.3.2 Procedural_solid_representation_sequence   

A Procedural_solid_representation_sequence is a type of Procedural_shape_representation_sequence that specifies a sequence of shape modification operations to create a Modified_solid. It is used to represent the construction history of a procedural solid model.

NOTE    This entity is incompletely modelled in the ARM. Complete specification can be obtained in the definition of the corresponding MIM entity.

EXPRESS specification:

*)
ENTITY Procedural_solid_representation_sequence
  SUBTYPE OF (Procedural_shape_representation_sequence);
END_ENTITY;
