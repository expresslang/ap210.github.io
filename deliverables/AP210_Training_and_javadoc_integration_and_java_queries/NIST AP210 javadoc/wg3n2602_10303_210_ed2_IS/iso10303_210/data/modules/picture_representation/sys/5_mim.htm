5.2 MIM EXPRESS short listing

This clause specifies the EXPRESS schema derived from the mapping table. It uses elements from the common resources or from other application modules and defines the EXPRESS constructs that are specific to this part of ISO 10303.

This clause constitutes the Module Interpreted Module (MIM) of the application module.

This clause also specifies the modifications that apply to the constructs imported from the common resources.

The following restrictions apply to the use, in this schema, of constructs defined in common resources or in application modules:

  • use of a supertype entity does not make applicable any of its specializations, unless the specialization is also imported in the MIM schema;
  • use of a SELECT type does not make applicable any of its listed types unless the listed type is also imported in the MIM schema.

EXPRESS specification:

*)

5.2.2 MIM type definitions

This subclause specifies the MIM types for this application module. The MIM types and definitions are specified below.

5.2.2.1 picture_representation_item_select   

The picture_representation_item_select type allows for the designation of the data types axis2_placement_2d, picture_representation_item, planar_box and styled_item.

EXPRESS specification:

*)
TYPE picture_representation_item_select = SELECT
   (axis2_placement_2d,
    picture_representation_item,
    planar_box,
    styled_item);
END_TYPE;
(*

5.2.2.2 pr_external_identification_item   

The pr_external_identification_item type is an extension of the external_identification_item type. It adds the data type externally_defined_picture_representation_item to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE pr_external_identification_item = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON external_identification_item WITH
   (externally_defined_picture_representation_item);
END_TYPE;
(*

5.2.3 MIM entity definitions

This subclause specifies the MIM entities for this module. The MIM entities and definitions are specified below.

5.2.3.1 externally_defined_picture_representation_item   

An externally_defined_picture_representation_item is a type of picture_representation_item that implements the ARM concept of Externally_defined_picture_representation_item.

EXPRESS specification:

*)
ENTITY externally_defined_picture_representation_item
  SUBTYPE OF (picture_representation_item);
INVERSE
  source : applied_external_identification_assignment FOR items;
WHERE
  WR1: NOT (SELF\representation_item.name IN pre_defined_picture_representation_types);
END_ENTITY;
(*

Attribute definitions:

source: an inverse relationship that specifies that the existence of the externally_defined_picture_representation_item is dependent on the existence of the applied_external_identification_assignment that specifies the externally_defined_picture_representation_item as its items.

Formal propositions:

WR1: A name of externally_defined_picture_representation_item shall not be from a set of names defined in pre_defined_picture_representation_types.

5.2.3.2 picture_representation   

A picture_representation is a type of presentation_view that implements the ARM concept of Picture_representation.

EXPRESS specification:

*)
ENTITY picture_representation
  SUBTYPE OF (presentation_view);
  SELF\representation.items : SET[2:?] OF picture_representation_item_select;
INVERSE
  size : presentation_size FOR unit;
WHERE
  WR1: SIZEOF(QUERY(items <* SELF\representation.items | 'PICTURE_REPRESENTATION_MIM.PICTURE_REPRESENTATION_ITEM' IN TYPEOF(items))) = 1;
END_ENTITY;
(*

Attribute definitions:

items: an attribute inherited from the representation shall be redeclared as the picture_representation_item_select for the picture_representation. There shall exist two or more picture_representation_item_select for the picture_representation.

size: an inverse relationship that specifies that the existence of the picture_representation is dependent on the existence of the presentation_size that specifies the picture_representation as its unit.

Formal propositions:

WR1: A picture_representation must have exactly one picture_representation_item as an item in the items list.

5.2.3.3 picture_representation_item   

A picture_representation_item is a type of bytes_representation_item that implements the ARM concept of Picture_representation_item. A picture_representation_item is either an externally_defined_picture_representation_item or a predefined_picture_representation_item.

EXPRESS specification:

*)
ENTITY picture_representation_item
  ABSTRACT SUPERTYPE OF (ONEOF (externally_defined_picture_representation_item,
                                predefined_picture_representation_item))
  SUBTYPE OF (bytes_representation_item);
END_ENTITY;
(*

5.2.3.4 predefined_picture_representation_item   

A predefined_picture_representation_item is a type of picture_representation_item that implements the ARM concept of Predefined_picture_representation_item.

EXPRESS specification:

*)
ENTITY predefined_picture_representation_item
  SUBTYPE OF (picture_representation_item);
WHERE
  WR1: SELF\representation_item.name IN pre_defined_picture_representation_types;
END_ENTITY;
(*

Formal propositions:

WR1: A name of predefined_picture_representation_item shall be from a set of names defined in pre_defined_picture_representation_types.



*)
END_SCHEMA;  -- Picture_representation_mim
