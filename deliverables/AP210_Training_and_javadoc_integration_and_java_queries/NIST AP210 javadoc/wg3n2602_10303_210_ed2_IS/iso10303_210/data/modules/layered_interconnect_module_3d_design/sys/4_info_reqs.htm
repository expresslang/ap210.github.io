4 Information requirements

This clause specifies the information requirements for the Layered interconnect module 3D design application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Layered_interconnect_module_3d_design_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Layered_interconnect_module_3d_design_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Physical_unit_3d_design_view_arm;    --  ISO/TS 10303-1725

USE FROM Stratum_non_planar_shape_arm;    --  ISO/TS 10303-1748

REFERENCE FROM Characterizable_object_arm    --  ISO/TS 10303-1765
  (bag_to_set);
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Physical_unit_3d_design_view_arm ISO/TS 10303-1725
Stratum_non_planar_shape_arm ISO/TS 10303-1748
Characterizable_object_arm ISO/TS 10303-1765

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.2.1 lim3dd_sf_shape_select   

The lim3dd_sf_shape_select type is an extension of the limd_sf_shape_select type. It adds the data type Solid_model to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE lim3dd_sf_shape_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON limd_sf_shape_select WITH
   (Solid_model);
END_TYPE;
(*

4.2.2 lim3dd_stratum_shape_model_select   

The lim3dd_stratum_shape_model_select type is an extension of the limd_stratum_shape_model_select type. It adds the data type Stratum_3d_shape_model to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE lim3dd_stratum_shape_model_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON limd_stratum_shape_model_select WITH
   (Stratum_3d_shape_model);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Located_3d_inter_stratum_extent   

A Located_3d_inter_stratum_extent is a type of Inter_stratum_extent that provides the line in three dimensional space along which the Inter_stratum_extent was determined. The line is provided by a location and a direction.

EXPRESS specification:

*)
ENTITY Located_3d_inter_stratum_extent
  SUBTYPE OF (Inter_stratum_extent);
  location : Cartesian_point;
  direction : Direction;
END_ENTITY;
(*

Attribute definitions:

location: specifies the role of a member of Cartesian_point for the Inter_stratum_extent. There is no requirement for the location to be in the domain of a specific Stratum.

direction: the role of a member of Direction for the Located_3d_inter_stratum_extent.

NOTE    Because more than one Stratum_technology_occurrence_link may reference the same Stratum_technology_occurrence, in some cases it will be impossible to algorithmically determine a specific path through the stack without knowing the line along which the sequence is determined. Therefore the pre-processor shall populate members of Located_3d_inter_stratum_extent when necessary to disambiguate the sequence of stratum in a three dimensional context. The purpose is only to establish an unambiguous line in three dimensional space and is not intended to specify starting and terminus conditions. from_stratum and to_stratum inherited from Inter_stratum_extent establish starting and terminus conditions.

4.3.2 Stratum_3d_shape_model   

A Stratum_3d_shape_model is a type of Geometric_model. A Stratum_3d_shape_model is the three-dimensional shape of a Stratum.

EXPRESS specification:

*)
ENTITY Stratum_3d_shape_model
  SUBTYPE OF (Geometric_model);
  shape_characterized_component : Stratum;
WHERE
  WR1: NOT ('LAYERED_INTERCONNECT_MODULE_3D_DESIGN_ARM.PLANAR_SHAPE_MODEL' IN TYPEOF(SELF));
  WR2: NOT EXISTS(SELF\Representation.name);
  WR3: NOT EXISTS(SELF\Representation.description);
  WR4: SIZEOF(USEDIN(SELF, 'SHAPE_PROPERTY_ASSIGNMENT_ARM.SHAPE_DESCRIPTION_ASSOCIATION.REPRESENTATION'))=0;
END_ENTITY;
(*

Attribute definitions:

shape_characterized_component: specifies a role of the Stratum for the Stratum_3d_shape_model.

Formal propositions:

WR1: The Stratum_3d_shape_model shall not be a Planar_shape_model.

WR2: The name shall not be populated.

WR3: The description shall not be populated.

WR4: The Stratum_3d_shape_model shall not be used in role of representation by any Shape_description_association.

4.3.3 Stratum_interconnect_module_3d_position   

A Stratum_interconnect_module_3d_position is the position of the Stratum model in the Interconnect_module_design_view model in three dimensions.

EXPRESS specification:

*)
ENTITY Stratum_interconnect_module_3d_position;
  stratum_component : Interconnect_module_stratum_assembly_relationship;
  stratum_shape : Stratum_3d_shape_model;
  interconnect_module_shape : Physical_unit_3d_shape_model;
  transformation : Axis_placement_3d;
UNIQUE
  UR1: stratum_component;
END_ENTITY;
(*

Attribute definitions:

stratum_component: specifies a role of the Interconnect_module_stratum_assembly_relationship for the Stratum_interconnect_module_3d_position.

stratum_shape: specifies the Stratum_3d_shape_model for the Stratum model for the Stratum_interconnect_module_3d_position.

interconnect_module_shape: specifies the Physical_unit_3d_shape_model that is the Interconnect_module_design_view model for the Stratum_interconnect_module_3d_position.

transformation: specifies a role of the Axis_placement_3d for the Stratum_interconnect_module_3d_position.

Formal propositions:

UR1: The stratum_component shall be unique within a population of Stratum_interconnect_module_3d_positions.

4.4 ARM function definitions

This subclause specifies the ARM functions for this module. The ARM functions and definitions are specified below.

4.4.1 in_3d_assembly_context

The in_3d_assembly_context function returns TRUE only if specified Area_component has at least one Assembly_component_3d_shape_model referencing it as shape_characterized_component and if all such Assembly_component_3d_shape_models share context_of_items with at least one Physical_unit_3d_shape_model of assembly they are assembled into. Otherwise the function returns FALSE.

EXPRESS specification:

*)
FUNCTION in_3d_assembly_context (comp : Assembly_component) :LOGICAL; 
LOCAL
    shapes : SET OF Assembly_component_3d_shape_model := [];
    shapes_in_context : SET OF Physical_unit_3d_shape_model := [];
    assemblies : SET OF Product_view_definition := [];
    i, j : INTEGER := 0;
    in_context : LOGICAL := TRUE;
  END_LOCAL;
  
  REPEAT i := 1 TO SIZEOF(comp.occurrence_contexts);
    assemblies := assemblies + comp.occurrence_contexts[i].relating_view;
  END_REPEAT;

  shapes := bag_to_set(USEDIN(comp,'PHYSICAL_UNIT_3D_DESIGN_VIEW_ARM.ASSEMBLY_COMPONENT_3D_SHAPE_MODEL.SHAPE_CHARACTERIZED_COMPONENT'));
  REPEAT i := 1 TO SIZEOF(shapes);
    shapes_in_context := bag_to_set(USEDIN(shapes[i]\Geometric_model.context_of_items,'ELEMENTAL_GEOMETRIC_SHAPE_ARM.GEOMETRIC_MODEL.CONTEXT_OF_ITEMS'));
    in_context := FALSE;
    j := 1;
    REPEAT WHILE ((j <= SIZEOF(shapes_in_context)) AND (NOT (in_context)));
      in_context := (shapes_in_context[j].shape_characterized_definition IN assemblies);
      j := j + 1;
    END_REPEAT;
    
    IF NOT (in_context) THEN
      RETURN (FALSE);
    END_IF;
  END_REPEAT;
  
  RETURN (SIZEOF(shapes) > 0);
END_FUNCTION;
(*

Argument definitions:

comp: the specified Assembly_component.

4.4.2 no_3d_location_for_component

The no_3d_location_for_component function returns TRUE only if there is no Component_3d_location given for the specified component. Otherwise it returns FALSE.

EXPRESS specification:

*)
FUNCTION no_3d_location_for_component (comp : Assembly_component) :BOOLEAN; 
LOCAL
    i, j : INTEGER := 0;
    cis : SET OF Contextual_item_shape := [];
    loc : SET OF Component_3d_location := [];
  END_LOCAL;
  
  REPEAT i := 1 TO SIZEOF(comp.occurrence_contexts);
    cis := bag_to_set(USEDIN(comp.occurrence_contexts[i],'SHAPE_PROPERTY_ASSIGNMENT_ARM.CONTEXTUAL_ITEM_SHAPE.DESCRIBED_ELEMENT'));
    REPEAT j := 1 TO SIZEOF(cis);
      loc := bag_to_set(USEDIN(cis[j],'CONTEXTUAL_SHAPE_POSITIONING_ARM.CONTEXTUAL_SHAPE_REPRESENTATION.CONTEXTUAL_SHAPE'));
      IF (SIZEOF(loc) > 0) THEN
        RETURN (FALSE);
      END_IF;
    END_REPEAT;
  END_REPEAT;

  RETURN (TRUE);
END_FUNCTION;
(*

Argument definitions:

comp: the specified Assembly_component.

4.5 ARM rule definitions

This subclause specifies the ARM rules for this module. The ARM rules and definitions are specified below.

4.5.1 component_in_assembly_3d_context

The component_in_assembly_3d_context constrains each Area_component to have at least one Assembly_component_3d_shape_model referencing it as shape_characterized_component and if all such Assembly_component_3d_shape_models share context_of_items with at least one Physical_unit_3d_shape_model of assembly they are assembled into.

EXPRESS specification:

*)
RULE component_in_assembly_3d_context FOR
(Area_component);
WHERE
  WR1: SIZEOF(QUERY( ac <* Area_component | NOT( (SIZEOF(USEDIN(ac,'PHYSICAL_UNIT_3D_DESIGN_VIEW_ARM.ASSEMBLY_COMPONENT_3D_SHAPE_MODEL.SHAPE_CHARACTERIZED_COMPONENT')) = 0) OR (in_3d_assembly_context(ac))) )) = 0;
END_RULE;
(*

Argument definitions:

Area_component : the set of all instances of Area_component.

Formal propositions:

WR1: Returns TRUE if each Area_component has at least one Assembly_component_3d_shape_model referencing it as shape_characterized_component and if all such Assembly_component_3d_shape_models share context_of_items with at least one Physical_unit_3d_shape_model of assembly they are assembled into.

4.5.2 no_component_assembly_3d_location_for_area_component

The no_component_assembly_3d_location_for_area_component constrains no each Area_component to prohibit it from being located in 3D by a Component_3d_location.

NOTE    An Area_component is geometrically defined in place.

EXPRESS specification:

*)
RULE no_component_assembly_3d_location_for_area_component FOR
(Area_component);
WHERE
  WR1: SIZEOF(QUERY( ac <* Area_component | NOT( (SIZEOF(USEDIN(ac,'PHYSICAL_UNIT_3D_DESIGN_VIEW_ARM.ASSEMBLY_COMPONENT_3D_SHAPE_MODEL.SHAPE_CHARACTERIZED_COMPONENT')) = 0) OR (no_3d_location_for_component(ac))) )) = 0;
END_RULE;
(*

Argument definitions:

Area_component : the set of all instances of Area_component.

Formal propositions:

WR1: Each Area_component shall not be referenced by any Assembly_component_3d_shape_model as shape_characterized_component.



*)
END_SCHEMA;  -- Layered_interconnect_module_3d_design_arm
