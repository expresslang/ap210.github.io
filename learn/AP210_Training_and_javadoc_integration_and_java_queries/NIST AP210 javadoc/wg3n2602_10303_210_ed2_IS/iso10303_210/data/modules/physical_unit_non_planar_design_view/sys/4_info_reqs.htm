4 Information requirements

This clause specifies the information requirements for the Physical unit non planar design view application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Physical_unit_non_planar_design_view_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Physical_unit_non_planar_design_view_arm;

4.2 ARM entity definition

This subclause specifies the ARM entity for this module. The ARM entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entity and definition is specified below.

4.2.1 Assembly_component_non_planar_shape_model   

An Assembly_component_non_planar_shape_model is a type of Manifold_subsurface_model. An Assembly_component_non_planar_shape_model is the shape of a Assembly_component that is represented in the form of a surface.

EXPRESS specification:

*)
ENTITY Assembly_component_non_planar_shape_model
  SUBTYPE OF (Manifold_subsurface_model);
  shape_characterized_component : SET[1:?] OF Assembly_component;
DERIVE
  equivalent_reference_shape_model : SET[0:1] OF Geometric_model_relationship := bag_to_set(QUERY( gmr <* USEDIN(SELF, 'EXTENDED_BASIC_GEOMETRY_ARM.GEOMETRIC_MODEL_RELATIONSHIP.REP_2') | (gmr\Representation_relationship.relation_type = 'equivalent reference shape model')));
WHERE
  WR1: NOT EXISTS(SELF\Representation.description);
  WR2: SIZEOF(equivalent_reference_shape_model) < 2;
END_ENTITY;
(*

Attribute definitions:

shape_characterized_component: a set of one or more Assembly_components for the Assembly_component_non_planar_shape_model.

equivalent_reference_shape_model: specifies the reference shape model for the present AO. The equivalent_reference_shape_model is to be provided only if the type of the shape representation of the assembly is different from the type of the shape representation of the part and establishes an equivalency relationship between the shape representation of the part and that of the assembly.

Formal propositions:

WR1: The description shall not be populated.

WR2: No more than one equivalent_reference_shape_model shall be provided.

Informal propositions:

IP1: If the type of the shape representation of the part is the same as that of the assembly the equivalent_reference_shape_model shall not be provided.

4.3 ARM rule definition

This subclause specifies the ARM rule for this module. The ARM rule and definition is specified below.

4.3.1 unique_assembly_component_non_planar_shape_model

The unique_assembly_component_non_planar_shape_model rule constrains the population of Assembly_component_non_planar_shapes, so that there shall be no more than one Assembly_component_non_planar_shape_model for each Assembly_component.

EXPRESS specification:

*)
RULE unique_assembly_component_non_planar_shape_model FOR
(Assembly_component_non_planar_shape_model);
LOCAL
     ac : BAG OF Assembly_component := [];
   END_LOCAL;
   REPEAT i := 1 to SIZEOF(Assembly_component_non_planar_shape_model) by 1;
     ac  := ac + Assembly_component_non_planar_shape_model[i].shape_characterized_component;
   END_REPEAT;
WHERE
  WR1: SIZEOF(ac) = SIZEOF(bag_to_set(ac));
END_RULE;
(*

Argument definitions:

Assembly_component_non_planar_shape_model : the set of all instances of Assembly_component_non_planar_shape_model.

Formal propositions:

WR1: There shall be no more than one Assembly_component_non_planar_shape_model for each Assembly_component.



*)
END_SCHEMA;  -- Physical_unit_non_planar_design_view_arm
