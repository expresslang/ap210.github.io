4 Information requirements

This clause specifies the information requirements for the Chemical substance application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Chemical_substance_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Chemical_substance_arm;
(*

4.3 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.3.1 cs_material_item_select   

The cs_material_item_select type is an extension of the material_item_select type. It adds the data type Substance_view_definition to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE cs_material_item_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON material_item_select WITH
   (Substance_view_definition);
END_TYPE;
(*

4.3.2 substance_composition_basis   

The substance_composition_basis type specifies the basis on which a substance is decomposed into its constituents.

EXPRESS specification:

*)
TYPE substance_composition_basis = EXTENSIBLE ENUMERATION OF
   (VOLUME,
    MASS,
    MOLES,
    ATOMS);
END_TYPE;
(*

Enumerated item definitions:

VOLUME: the composition is based on the volume ratio between a constituent and the composition.

MASS: the composition is based on the weight ratio between a constituent and the composition.

MOLES: the composition is based on the mole ratio between a constituent and the composition.

ATOMS: the composition is based on the atom ratio between a constituent and the composition.

4.4 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.4.1 Chemical_compound   

A Chemical_compound is a type of Substance that represents a chemical compound

EXPRESS specification:

*)
ENTITY Chemical_compound
  SUBTYPE OF (Substance);
DERIVE
  id_owner : SET[0:?] OF Organization_or_person_in_organization_assignment := QUERY(temp <* bag_to_set(USEDIN(SELF,'PERSON_ORGANIZATION_ASSIGNMENT_ARM.'+ 'ORGANIZATION_OR_PERSON_IN_ORGANIZATION_ASSIGNMENT.ITEMS')) | (temp.role = 'id owner') );
WHERE
  WR1: EXISTS (id_owner) AND (SIZEOF(id_owner) = 1);
END_ENTITY;
(*

Attribute definitions:

id_owner: specifies the Organization_or_person_in_organization_assignment that refers this Chemical_compound as a member of items and has role set to 'id owner'. Where applicable the id_owner shall be an organization with the name "CAS registry".

Formal propositions:

WR1: The Chemical_compound shall belong to a product category named 'chemical compound'.

4.4.2 Chemical_element   

A Chemical_element is a type of Substance that represents a chemical element

EXPRESS specification:

*)
ENTITY Chemical_element
  SUBTYPE OF (Substance);
END_ENTITY;
(*

4.4.3 Physical_particle   

A Physical_particle is a type of Substance that represents a physical particle

EXPRESS specification:

*)
ENTITY Physical_particle
  SUBTYPE OF (Substance);
END_ENTITY;
(*

4.4.4 Raw_material_definition_relationship   

A Raw_material_definition_relationship is a type of View_definition_usage. A Raw_material_definition_relationship is the association between a Part_view_definition and a Substance_view_definition the part is made of.

EXPRESS specification:

*)
ENTITY Raw_material_definition_relationship
  SUBTYPE OF (View_definition_usage);
  SELF\View_definition_relationship.relating_view RENAMED application : Part_view_definition;
  SELF\View_definition_relationship.related_view RENAMED definition : Substance_view_definition;
UNIQUE
  UR1: application;
WHERE
  WR1: NOT EXISTS(SELF\View_definition_relationship.relation_type);
END_ENTITY;
(*

Attribute definitions:

application: the Part_view_definition that specifies the definition of the part version that is made from the substance version.

definition: the Substance_view_definition that specifies the definition of the substance version the part version is made of.

Formal propositions:

UR1: There shall be at most one Raw_material_definition_relationship for a Part_view_definition in the role of application.

WR1: The relation_type shall not be populated.

4.4.5 Substance   

A Substance is a type of Product that collects the definitional information of the versions of a substance. A Substance may be one of a Chemical_element, Chemical_compound or a Physical_particle.

EXPRESS specification:

*)
ENTITY Substance
  SUPERTYPE OF (ONEOF (Chemical_element,
                       Chemical_compound,
                       Physical_particle))
  SUBTYPE OF (Product);
UNIQUE
  UR1: SELF\Product.id;
END_ENTITY;
(*

Formal propositions:

UR1: The id shall be unique within a population of Substance.

4.4.6 Substance_composition_relationship   

An Substance_composition_relationship is a type of View_definition_relationship that relates two Substance_view_definitions for the purpose of specifying that one is a constituent of the other. The amount of the composition is given as a ratio based on volume, mass, moles or atoms of the whole Substance_view_definition.

EXPRESS specification:

*)
ENTITY Substance_composition_relationship
  SUBTYPE OF (View_definition_relationship);
  SELF\View_definition_relationship.related_view RENAMED constituent : Substance_view_definition;
  SELF\View_definition_relationship.relating_view RENAMED whole : Substance_view_definition;
  composition_basis : substance_composition_basis;
  determination_method : STRING;
  amount : Value_with_unit;
WHERE
  WR1: 'VALUE_WITH_UNIT_ARM.RATIO_UNIT' IN TYPEOF(amount.unit);
  WR2: NOT EXISTS(SELF\View_definition_relationship.relation_type);
END_ENTITY;
(*

Attribute definitions:

constituent: specifies the Substance_view_definition for which the ratio given by amount applies to.

whole: specifies the Substance_view_definition that serves as 100% for the ratio given by amount.

composition_basis: specifies the substance_composition_basis, the kind of ratio given by amount.

determination_method: specifies the LOGICAL type Substance_composition_relationship. This type permits a user to specify that the amount of material to be supplied with the product may be determined at assembly on an item by item basis, where the item referred to is a real product delivered to a customer.

EXAMPLE    Adhesives, insulation and wire may all be cases where the value of the attribute is TRUE.

amount: specifies a role of the Value_with_unit for the Substance_composition_relationship.

Formal propositions:

WR1: The unit of amount shall be Ratio_unit.

WR2: The relation_type shall not be populated.

4.4.7 Substance_version   

A Substance_version is a type of Product_version that identifies a version of a substance. A Substance_version serves as the collector of the data characterizing a substance.

EXPRESS specification:

*)
ENTITY Substance_version
  SUBTYPE OF (Product_version);
  SELF\Product_version.of_product : Substance;
END_ENTITY;
(*

Attribute definitions:

of_product: the Substance of which the Substance_version specifies a version.

4.4.8 Substance_view_definition   

A Substance_view_definition is a type of Product_view_definition that defines a characterization view of a version of a Substance.

EXPRESS specification:

*)
ENTITY Substance_view_definition
  SUBTYPE OF (Product_view_definition);
  SELF\Product_view_definition.defined_version : Substance_version;
END_ENTITY;
(*

Attribute definitions:

defined_version: the Substance_version of which the Substance_view_definition defines a view.

4.5 ARM rule definitions

This subclause specifies the ARM rules for this module. The ARM rules and definitions are specified below.

4.5.1 chemical_element_constraint

The rule chemical_element_constraint constrains the use of Product as name to Substance.

EXPRESS specification:

*)
RULE chemical_element_constraint FOR
(Product);
WHERE
  WR1: SIZEOF ( QUERY ( p <* Product | (('CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE' IN TYPEOF (p) ) AND ((p\Product.name) IN chemical_elements) ) OR (NOT('CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE' IN TYPEOF (p) ) AND NOT((p\Product.name) IN chemical_elements) ))) = SIZEOF ( Product);
END_RULE;
(*

Argument definitions:

Product : the set of all instances of Product.

Formal propositions:

WR1: A Product shall be a Substance if and only if the Product.name is in the list of elements specified in ISO 31-9, Annex A.

4.5.2 substance_version_constraint

The rule substance_version_constraint constrains the use of Substance as of_product to Substance_version only.

EXPRESS specification:

*)
RULE substance_version_constraint FOR
(Product_version);
WHERE
  WR1: SIZEOF ( QUERY ( pv <* Product_version | ( NOT ( 'CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE_VERSION' IN TYPEOF ( pv) ) ) AND ( 'CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE' IN TYPEOF ( pv.of_product) ) ) ) =0;
END_RULE;
(*

Argument definitions:

Product_version : the set of all instances of Product_version.

Formal propositions:

WR1: There shall not be any instance of Product_version that refers Substance as of_product and is not an instance of Substance_version.

4.5.3 substance_view_definition_constraint

The rule substance_view_definition_constraint constrains the use of Substance_version as defined_version to Substance_view_definition only.

EXPRESS specification:

*)
RULE substance_view_definition_constraint FOR
(Product_view_definition);
WHERE
  WR1: SIZEOF ( QUERY ( pvd <* Product_view_definition | ( NOT ( 'CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE_VIEW_DEFINITION' IN TYPEOF (pvd) ) ) AND ( 'CHEMICAL_SUBSTANCE_ARM.' + 'SUBSTANCE_VERSION' IN TYPEOF ( pvd.defined_version) ) ) ) =0;
END_RULE;
(*

Argument definitions:

Product_view_definition : the set of all instances of Product_view_definition.

Formal propositions:

WR1: There shall not be any instance of Product_view_definition that refers Substance_version as defined_version and is not an instance of Substance_view_definition.



*)
END_SCHEMA;  -- Chemical_substance_arm
