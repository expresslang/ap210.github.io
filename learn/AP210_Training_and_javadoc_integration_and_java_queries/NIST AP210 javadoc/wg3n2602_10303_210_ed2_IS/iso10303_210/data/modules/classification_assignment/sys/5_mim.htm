5.2 MIM EXPRESS short listing

This clause specifies the EXPRESS schema derived from the mapping table. It uses elements from the common resources or from other application modules and defines the EXPRESS constructs that are specific to this part of ISO 10303.

This clause constitutes the Module Interpreted Module (MIM) of the application module.

This clause also specifies the modifications that apply to the constructs imported from the common resources.

The following restrictions apply to the use, in this schema, of constructs defined in common resources or in application modules:

  • use of a supertype entity does not make applicable any of its specializations, unless the specialization is also imported in the MIM schema;
  • use of a SELECT type does not make applicable any of its listed types unless the listed type is also imported in the MIM schema.

5.2.1 MIM type definition

This subclause specifies the MIM type for this application module. The MIM type and definition is specified below.

5.2.1.1 classification_item   

The classification_item type is an extensible list of alternate data types. Additional alternate data types are specified in select data types that extend the classification_item type.

NOTE   This empty extensible select requires extension in a further module to ensure that entities that refer to it have at least one valid instantiation.

EXPRESS specification:

*)
TYPE classification_item = EXTENSIBLE GENERIC_ENTITY SELECT;
END_TYPE;
(*

5.2.2 MIM entity definition

This subclause specifies the MIM entity for this module. The MIM entity and definition is specified below.

5.2.2.1 applied_classification_assignment   

An applied_classification_assignment is a type of classification_assignment that associates an instance of group or of one of its specializations with one or more classification_item instances for classification purposes.

The considered group is specified with the inherited attribute assigned_class.

NOTE 1   The considered group may be a class or a characterized_object.

NOTE 2   Specific application modules address particular kinds of classification.

EXAMPLE    The entity applied_security_classification_assignment will be used in order to represent confidentiality level classification.

The meaning of the association is specified with the inherited attribute role.

Where applicable, the following value of classification role name shall be used:

  • 'reference': the items to which the classification assignment applies, are by intent incompletely transferred.

NOTE 3   This value can be used when assembly data are exchanged iteratively between two partners, to avoid to send twice the CAD files that contain the geometric representation of each component.

EXPRESS specification:

*)
ENTITY applied_classification_assignment
  SUBTYPE OF (classification_assignment);
  items : SET[1:?] OF classification_item;
END_ENTITY;
(*

Attribute definitions:

items: the set of one or more instances of types listed in classification_item, that are associated with the classification information.



*)
