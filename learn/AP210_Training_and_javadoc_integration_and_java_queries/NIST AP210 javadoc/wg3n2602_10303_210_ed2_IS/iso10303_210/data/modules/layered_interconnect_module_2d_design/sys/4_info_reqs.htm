4 Information requirements

This clause specifies the information requirements for the Layered interconnect module 2D design application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Layered_interconnect_module_2d_design_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Layered_interconnect_module_2d_design_arm;
(*

4.1 Required AM ARM

The following EXPRESS interface statement specifies the elements imported from the ARM of another application module.

EXPRESS specification:

*)
USE FROM Layered_interconnect_module_design_arm;    --  ISO/TS 10303-1698
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Layered_interconnect_module_design_arm ISO/TS 10303-1698

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM type definitions

This subclause specifies the ARM types for this application module. The ARM types and definitions are specified below.

4.2.1 lim2dd_sf_shape_select   

The lim2dd_sf_shape_select type is an extension of the limd_sf_shape_select type. It adds the data type Primitive_2d to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE lim2dd_sf_shape_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON limd_sf_shape_select WITH
   (Primitive_2d);
END_TYPE;
(*

4.2.2 lim2dd_stratum_shape_model_select   

The lim2dd_stratum_shape_model_select type is an extension of the limd_stratum_shape_model_select type. It adds the data type Stratum_planar_shape_model to the list of alternate data types.

NOTE  The list of entity data types may be extended in application modules that use the constructs of this module.

EXPRESS specification:

*)
TYPE lim2dd_stratum_shape_model_select = EXTENSIBLE GENERIC_ENTITY SELECT BASED_ON limd_stratum_shape_model_select WITH
   (Stratum_planar_shape_model);
END_TYPE;
(*

4.3 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.3.1 Located_2d_inter_stratum_extent   

A Located_2d_inter_stratum_extent is a type of Inter_stratum_extent that provides the location where the Inter_stratum_extent was determined.

EXPRESS specification:

*)
ENTITY Located_2d_inter_stratum_extent
  SUBTYPE OF (Inter_stratum_extent);
  location : Cartesian_point;
END_ENTITY;
(*

Attribute definitions:

location: specifies the role of a member of Cartesian_point for the Inter_stratum_extent.

NOTE    Because more than one Stratum_technology_occurrence_link may reference the same Stratum_technology_occurrence, in some cases it will be impossible to algorithmically determine a specific path through the stack without knowing the location. Therefore the pre-processor shall populate members of Located_2d_inter_stratum_extent when necessary to disambiguate the sequence of stratum.

4.3.2 Stratum_interconnect_module_planar_position   

A Stratum_interconnect_module_planar_position is the position of the Stratum model in the Interconnect_module_design_view model in two dimensions.

EXPRESS specification:

*)
ENTITY Stratum_interconnect_module_planar_position;
  stratum_component : Interconnect_module_stratum_assembly_relationship;
  stratum_shape : Stratum_planar_shape_model;
  interconnect_module_shape : Physical_unit_planar_shape_model;
  transformation : Axis_placement_2d;
UNIQUE
  UR1: stratum_component;
END_ENTITY;
(*

Attribute definitions:

stratum_component: specifies a role of the Interconnect_module_stratum_assembly_relationship for the Stratum_interconnect_module_planar_position.

stratum_shape: specifies the Stratum_planar_shape_model for the Stratum model for the Stratum_interconnect_module_planar_position.

interconnect_module_shape: specifies the Physical_unit_planar_shape_model for the Interconnect_module_design_view model for the Stratum_interconnect_module_planar_position.

transformation: specifies a role of the Axis_placement_2d for the Stratum_interconnect_module_planar_position.

Formal propositions:

UR1: The stratum_component shall be unique within a population of Stratum_interconnect_module_planar_positions.

4.3.3 Stratum_planar_shape_model   

A Stratum_planar_shape_model is a type of the Csg_2d_model. A Stratum_planar_shape_model is the two-dimensional shape model of a Stratum.

EXPRESS specification:

*)
ENTITY Stratum_planar_shape_model
  SUBTYPE OF (Csg_2d_model);
  shape_characterized_component : Stratum;
  SELF\Representation.items : SET[1:?] OF Primitive_2d;
UNIQUE
  UR1: shape_characterized_component;
WHERE
  WR1: NOT EXISTS(SELF\Representation.description);
END_ENTITY;
(*

Attribute definitions:

shape_characterized_component: specifies a role of the Stratum for the Stratum_planar_shape_model.

items: specifies the role of the set of Primitive_2d that are the shapes of the Stratum_features related to the Stratum specified by shape_characterized_component.

Formal propositions:

UR1: The shape_characterized_component shall be unique within a population of Stratum_planar_shape_models.

WR1: The description shall not be populated.



*)
END_SCHEMA;  -- Layered_interconnect_module_2d_design_arm
