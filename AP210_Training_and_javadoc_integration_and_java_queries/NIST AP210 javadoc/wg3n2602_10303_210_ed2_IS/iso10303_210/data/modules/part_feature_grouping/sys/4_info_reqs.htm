4 Information requirements

This clause specifies the information requirements for the Part feature grouping application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Part_feature_grouping_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Part_feature_grouping_arm;
(*

4.1 Required AM ARMs

The following EXPRESS interface statements specify the elements imported from the ARMs of other application modules.

EXPRESS specification:

*)
USE FROM Physical_unit_usage_view_arm;    --  ISO/TS 10303-1732

REFERENCE FROM Characterizable_object_arm    --  ISO/TS 10303-1765
  (bag_to_set);
(*

NOTE 1   The schemas referenced above are specified in the following part of ISO 10303:

Physical_unit_usage_view_arm ISO/TS 10303-1732
Characterizable_object_arm ISO/TS 10303-1765

NOTE 2   See Annex C, Figures C.1and C.2 for a graphical representation of this schema.

4.2 ARM entity definitions

This subclause specifies the ARM entities for this module. Each ARM application entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entities and definitions are specified below.

4.2.1 Part_array_feature   

A Part_array_feature is a type of Part_group_feature that is an array of Part_feature whose positions are specified as members of an array, and whose tolerances are applied based upon the array nature of Part_array_feature. A Part_array_feature is either a Part_linear_array_feature or a Part_rectangular_array_feature.

NOTE    Tolerances applied to Part_array_feature are specified by an object external to the Part_array_feature.

EXPRESS specification:

*)
ENTITY Part_array_feature
  ABSTRACT SUPERTYPE OF (ONEOF (Part_linear_array_feature,
                                Part_rectangular_array_feature))
  SUBTYPE OF (Part_group_feature);
END_ENTITY;
(*

4.2.2 Part_feature_placement_link   

A Part_feature_placement_link is a means to specify the order of Part_feature in a linear array.

EXPRESS specification:

*)
ENTITY Part_feature_placement_link;
  precedent_element : Part_feature;
  subsequent_element : Part_feature;
INVERSE
  referenced_by : Part_linear_array_feature FOR element;
WHERE
  WR1: acyclic_part_feature_placement_link(SELF, [subsequent_element], 'PART_FEATURE_GROUPING_ARM.PART_FEATURE_PLACEMENT_LINK');
END_ENTITY;
(*

Attribute definitions:

precedent_element: specifies a Part_feature for the Part_feature_placement_link.

subsequent_element: specifies a Part_feature for the Part_feature_placement_link.

referenced_by: an inverse relationship that specifies that the existence of the Part_feature_placement_link is dependent on the existence of the Part_linear_array_feature that specifies the Part_feature_placement_link as its element.

Formal propositions:

WR1: The graph of instances of the Application Object Part_feature_placement_link shall not contain a cycle.

4.2.3 Part_group_feature   

A Part_group_feature is a type of Part_feature and a type of Composite_shape_element that is an aggregation of Part_features. A Part_group_feature may be a Part_array_feature.

EXPRESS specification:

*)
ENTITY Part_group_feature
  SUPERTYPE OF (Part_array_feature)
  SUBTYPE OF (Composite_shape_element, Part_feature);
END_ENTITY;
(*

4.2.4 Part_linear_array_feature   

A Part_linear_array_feature is a type of Part_array_feature. The location_uncertainty refers to originating CAD system geometric uncertainty, not material processing tolerance.

EXPRESS specification:

*)
ENTITY Part_linear_array_feature
  SUBTYPE OF (Part_array_feature);
  element : SET[1:?] OF Part_feature_placement_link;
  initial_node_positive_x_displacement : Length_data_element;
  terminus_node_positive_x_displacement : Length_data_element;
  location_uncertainty : Length_data_element;
WHERE
  WR1: (1 + SIZEOF(element)) = pfg_get_sz_pf(element);
END_ENTITY;
(*

Attribute definitions:

element: specifies a role of the Part_feature_placement_link for the Part_linear_array_feature. There shall be one or more Part_feature_placement_link for a Part_linear_array_feature.

initial_node_positive_x_displacement: specifies the Length_data_element for the Part_linear_array_feature.

terminus_node_positive_x_displacement: specifies the Length_data_element for the Part_linear_array_feature.

location_uncertainty: specifies the Length_data_element for the Part_linear_array_feature.

Formal propositions:

WR1: The total number of Part_features referenced by members of elements shall be one more than the number of elements.

4.2.5 Part_linear_array_feature_link   

A Part_linear_array_feature_link is a means to associate two Part_linear_array_features.

EXPRESS specification:

*)
ENTITY Part_linear_array_feature_link;
  precedent_element : Part_linear_array_feature;
  subsequent_element : Part_linear_array_feature;
INVERSE
  referenced_by : Part_rectangular_array_feature FOR element;
WHERE
  WR1: acyclic_part_linear_array_feature_link(SELF, [subsequent_element], 'PART_FEATURE_GROUPING_ARM.PART_LINEAR_ARRAY_FEATURE_LINK');
END_ENTITY;
(*

Attribute definitions:

precedent_element: specifies the Part_linear_array_feature for the Part_linear_array_feature_link.

subsequent_element: specifies the Part_linear_array_feature for the Part_linear_array_feature_link.

referenced_by: an inverse relationship that specifies that the existence of the Part_linear_array_feature_link is dependent on the existence of the Part_rectangular_array_feature that specifies the Part_linear_array_feature_link as its element.

Formal propositions:

WR1: The graph of instances of the Application Object Part_linear_array_feature_link shall not contain a cycle.

4.2.6 Part_rectangular_array_feature   

A Part_rectangular_array_feature is a type of Part_array_feature. The location_uncertainty refers to originating CAD system uncertainty, not material processing tolerances.

EXPRESS specification:

*)
ENTITY Part_rectangular_array_feature
  SUBTYPE OF (Part_array_feature);
  element : SET[1:?] OF Part_linear_array_feature_link;
  initial_row_positive_y_displacement : Length_data_element;
  terminus_row_positive_y_displacement : Length_data_element;
  location_uncertainty : Length_data_element;
WHERE
  WR1: (1 + SIZEOF(element)) = pfg_get_sz_plar(element);
END_ENTITY;
(*

Attribute definitions:

element: specifies a role of the Part_linear_array_feature_link for the Part_rectangular_array_feature. There shall be one or more Part_linear_array_feature_link for a Part_rectangular_array_feature.

initial_row_positive_y_displacement: specifies the Length_data_element for the Part_rectangular_array_feature.

terminus_row_positive_y_displacement: specifies the Length_data_element for the Part_rectangular_array_feature.

location_uncertainty: specifies the Length_data_element for the Part_rectangular_array_feature.

Formal propositions:

WR1: The total number of Part_linear_array_features referenced by members of elements shall be one more than the number of elements.

4.3 ARM function definitions

This subclause specifies the ARM functions for this module. The ARM functions and definitions are specified below.

4.3.1 acyclic_part_feature_placement_link

The acyclic_part_feature_placement_link function determines whether the graph of instances of the Application Object type Part_feature_placement_link that contains relation as one of its links contains a cycle. This function may be used to evaluate Part_feature_placement_link or any of it's subtypes.

EXPRESS specification:

*)
FUNCTION acyclic_part_feature_placement_link (relation : Part_feature_placement_link; relatives : SET[1:?] OF Part_feature; specific_relation : STRING) :BOOLEAN; 
LOCAL
      x : SET OF Part_feature_placement_link := [];
    END_LOCAL;
    IF relation.precedent_element IN relatives THEN
      RETURN (FALSE);
    END_IF;
    x := QUERY(pd <* bag_to_set(USEDIN(relation.precedent_element,
      'PART_FEATURE_GROUPING_ARM.' +
      'PART_FEATURE_PLACEMENT_LINK.' +
      'SUBSEQUENT_ELEMENT')) | specific_relation IN TYPEOF(pd));
    REPEAT i := 1 TO HIINDEX(x);
      IF NOT acyclic_part_feature_placement_link(x[i], relatives +
        relation.precedent_element, specific_relation) THEN
        RETURN (FALSE);
      END_IF;
    END_REPEAT;
    RETURN (TRUE);
END_FUNCTION;
(*

Argument definitions:

relation: the specified Part_feature_placement_link.

relatives: the specified set of Part_feature.

specific_relation: the specified string.

4.3.3 pfg_get_sz_pf

The function pfg_get_sz_pfreturns the size of the set of ' Part_feature referenced by members of input.

EXPRESS specification:

*)
FUNCTION pfg_get_sz_pf (input : SET[1:?] OF Part_feature_placement_link) :INTEGER; 
LOCAL
      ac : SET OF Part_feature := [];
    END_LOCAL;
    REPEAT i := 1 TO HIINDEX(input);
      ac := ac + input[i]\Part_feature_placement_link.precedent_element +
        input[i]\Part_feature_placement_link.subsequent_element;
    END_REPEAT;
    RETURN (SIZEOF(ac));
END_FUNCTION;
(*

Argument definitions:

input: the specified set of Part_feature_placement_link.

4.3.4 pfg_get_sz_plar

The function pfg_get_sz_plar returns the size of the set of Part_linear_array_feature referenced by members of input.

EXPRESS specification:

*)
FUNCTION pfg_get_sz_plar (input : SET[1:?] OF Part_linear_array_feature_link) :INTEGER; 
LOCAL
      ac : SET OF Part_linear_array_feature := [];
    END_LOCAL;
    REPEAT i := 1 TO HIINDEX(input);
      ac := ac + input[i]\Part_linear_array_feature_link.precedent_element +
        input[i]\Part_linear_array_feature_link.subsequent_element;
    END_REPEAT;
    RETURN (SIZEOF(ac));
END_FUNCTION;
(*

Argument definitions:

input: the specified set of Part_linear_array_feature_link.
