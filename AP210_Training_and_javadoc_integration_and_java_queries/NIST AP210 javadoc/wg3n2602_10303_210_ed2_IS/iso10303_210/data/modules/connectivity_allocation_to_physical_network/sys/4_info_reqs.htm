4 Information requirements

This clause specifies the information requirements for the Connectivity allocation to physical network application module. The information requirements are specified as the Application Reference Model (ARM) of this application module.

NOTE 1  A graphical representation of the information requirements is given in Annex C.

NOTE 2  The mapping specification is specified in 5.1. It shows how the information requirements are met by using common resources and constructs defined or imported in the MIM schema of this application module.

The following EXPRESS specification begins the Connectivity_allocation_to_physical_network_arm schema and identifies the necessary external references.

EXPRESS specification:

*)
SCHEMA Connectivity_allocation_to_physical_network_arm;

4.2 ARM entity definition

This subclause specifies the ARM entity for this module. The ARM entity is an atomic element that embodies a unique application concept and contains attributes specifying the data elements of the entity. The ARM entity and definition is specified below.

4.2.1 Connectivity_allocation_to_physical_network   

A Connectivity_allocation_to_physical_network is a means to provide information about the implementation of a connectivity requirement to a network.

EXPRESS specification:

*)
ENTITY Connectivity_allocation_to_physical_network;
  implementation : Generic_physical_network;
  connectivity_requirement : Aggregate_connectivity_requirement;
UNIQUE
  UR1: implementation;
END_ENTITY;
(*

Attribute definitions:

implementation: specifies the Generic_physical_network for the Connectivity_allocation_to_physical_network.

connectivity_requirement: specifies the Aggregate_connectivity_requirement for the Connectivity_allocation_to_physical_network.

Formal propositions:

UR1: The implementation shall be unique within a population of Connectivity_allocation_to_physical_network.



*)
END_SCHEMA;  -- Connectivity_allocation_to_physical_network_arm
